# 🧠 知识图谱抽取技能 - 速查卡片

## 📋 快速使用参考

### 输入要求
| 文件类型 | 格式要求 | 说明 |
|----------|----------|------|
| 源文档 | `.docx` / `.pdf` | 含清晰结构的知识文档 |
| 模板文件 | `.xlsx` | A1单元格含格式规范 |

### 输出格式 (A-O列)
```
A | B | C | D | E | F | G | H | I | J | K | L | M | N | O
↑   ↑   ↑   ↑   ↑   ↑   ↑   ↑   ↑   ↑   ↑   ↑   ↑   ↑   ↑
1级 2级 3级 4级 5级 6级 7级 前 置 关 标 认 知 教 知识
级    级    级    级    级    级   置 后 联 签 知 识 学 点说
知    知    知    知    知    知  知 知 知     维 分 目 明  
识    识    识    识    识    识  点 点 点     度 类 标    
点    点    点    点    点    点                    
```

> **关键**: 每行 `A-G` 列只填**一个**知识点（树状结构）

## 📐 格式约束核心

| 约束 | 规定内容 | 检查方法 |
|------|----------|----------|
| **树状结构** | A-G列每行仅1非空单元格 | `SUM(A1:G1)=1` ✅ |
| **关系分隔** | H-I-J列用`;`分隔 | `电阻值;电流值` ✅ `电阻值；电流值` ❌ |
| **认知单选** | L列单选值 | 记忆/理解/应用/分析/评价/创造 |
| **分类单选** | M列单选值 | 事实性/概念性/程序性/元认知 |
| **模板完整** | 保留全部列 | 不删A-O列中任一列 |

### 认知层级关键词识别
| 识别词 | 认知层级 | 教育能力 |
|--------|----------|----------|
| 知道/了解/记住 | 记忆 | 识别、回忆 |
| 理解/阐明/说明 | 理解 | 解释、举例 |
| 掌握/能够/会 | 应用 | 任务执行 |
| 分析/比较/辨析 | 分析 | 关系拆解 |
| 评价/评估/判断 | 评价 | 价值衡量 |
| 设计/规划/创建 | 创造 | 新方案生成 |

## 🔧 核心命令

### 基础抽取命令
```bash
python scripts/extract_knowledge_graph.py \
  --source input_document.docx \
  --template assets/template.xlsx \
  --output output_result.xlsx
```

### 参数说明
| 参数 | 说明 | 示例 |
|------|------|------|
| `--source` | 源文档路径 | `/path/to/课程标准.docx` |
| `--template` | 格式模板 | `assets/template.xlsx` |
| `--output` | 输出路径 | `/path/to/output.xlsx` |

## 💡 智能关系识别

### H列 - 前置关系（需先掌握）
- 基础 → 高级（"学电容器需先懂电容概念"）
- 工具 → 应用（"先会用万用表后测电压"）
- 理论 → 实践（"知电学理后装电路"）

### I列 - 后置关系（学完后可学）
- 自动生成（H列的反向）

### J列 - 关联关系（相关而非依赖）
- 同域知识（"电压 ↔ 电流概念"）  
- 类别对比（"串连 ↔ 并连电路"）

## 🧪 验证检查表

### 输出验证要点
- [ ] A-G列：每行只有一个知识点
- [ ] H-J列：关系分隔符为英文`;`
- [ ] L列：认知维度为预设值之一
- [ ] M列：知识点分类为预设值之一
- [ ] 非空列：保持A-O列完整

### 质量检查命令
```bash
# 检查树状结构  
awk -F',' '{count=0; for(i=1;i<=7;i++) if($i != "") count++; if(count != 1) print "Error at line " NR, $0}' output.csv

# 检查关系分隔符
grep -n '[；，]' output.csv  # 检测中文分号/逗号（错误）
```

### 导入兼容性检查
- **Neo4j**: 
  - CSV头部字段映射正常
  - 关系节点匹配
- **Jena**:  
  - 数据类型匹配（字符串/数字）
  - 层级索引正确

## 🎯 优化建议

### 提升准确性
1. 使用 Word 标题样式而非手动编号
2. 在文档中加入认知层级标识词 
3. 清晰标记知识点间依赖关系词
4. 保持层级结构连续性（无跳跃）

### 文件准备策略
- **短小文档**: 建议 <50页，层次>5级
- **结构清晰**: 使用标题1-7样式分级
- **动词明确**: "了解/掌握/应用"等引导层级

## ⚠️ 关键提示

### 最常见的错误
1. **❌ 一行多知识点** → `['电路', '电阻', '欧姆',...]`  
2. **✅ 正确做法** → 每行一个，层级分多行
3. **❌ 中文分号** → `电阻值；电压值`  
4. **✅ 正确分隔** → `电阻值;电压值`

### 必需验证项
- ✅ 每行A-G列仅一个非空值
- ✅ H-J列使用英文`;`分隔  
- ✅ L/M列为预定义单选值
- ✅ 保留模板中的全部列格式
