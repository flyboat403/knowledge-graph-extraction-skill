# 快速入门：文档知识图谱抽取技能

## 1. 准备工作

### 1.1 环境检查
在开始使用本技能前，请确认：
- ✅ 已安装 pandoc (文档转换工具): `pandoc --version` 
- ✅ 已安装 openpyxl (Excel操作): `pip install openpyxl`
- ✅ 已安装 pandas (数据处理): `pip install pandas`

### 1.2 输入文件准备
准备两种类型的文件：

**A. 源文档** (待抽取知识点的文档) - 格式支持：
- `docx` - Word 文档 (推荐)
- `pdf` - PDF 文档 

**B. 模板文件** (定义输出格式要求) - 格式：
- `xlsx` - Excel 模板

模板 A1 单元格应包含格式规范（系统会自动读取）。

---

## 2. 执行抽取

### 2.1 使用命令行工具
```bash
# 使用示例
python scripts/extract_knowledge_graph.py \
  --source 课程标准.docx \
  --template template.xlsx \
  --output 输出结果.xlsx
```

**命令参数说明：**
- `--source`: 源文档路径 (必填)
- `--template`: 模板路径 (必填)  
- `--output`: 输出文件路径 (必填)
- `--dry-run`: 仅解析模板结构（选填）

### 2.2 处理流程演示
输入文档 `电子信息类职业技能考试大纲.pdf` 会自动输出为：


## 3. 理解输出结果

### 3.1 树状结构 (A-G列)
**记住**: 每行只填一个知识点！

✅ **正确**:
```
行1: [课程名称,   "",    "",    "",    "",    "",    ""]
行2: ["",     模块1,   "",    "",    "",    "",    ""]
行3: ["",     "",    单元1,   "",    "",    "",    ""]
```

❌ **错误（绝不可行）**:
```
['课程名称', '模块1', '单元1', '', '', '', '']  # 一行填多个节点
```

### 3.2 知识关系 (H-J列)
| 列 | 关系类型 | 含义示例 |
|----|----------|----------|
| H | 前置知识点 | "学习 A 前必须掌握 B、C" |
| I | 后置知识点 | "学完 A 后可继续学 D、E" |
| J | 关联知识点 | "A 与 F、G 有相关性但非前后" |

### 3.3 认知标注 (L-M列)
| L列(认知维度) | M列(知识分类) | 示例 |
|----------------|----------------|------|
| 记忆 | 事实性 | "说出电阻的单位" |
| 理解 | 概念性 | "解释欧姆定律含义" |
| 应用 | 程序性 | "使用万用表测量电压" |
| 分析 | 程序性 | "分析电路故障原因" |
| 评价 | 元认知 | "比较不同电路方案" |
| 创造 | 元认知 | "设计电源电路" |

## 4. 常见问题解答

### Q: 如何准备最优化输入文档？
**A**: 使用 Word 标题样式而非手动编号：

```
✅ 正确做法:
【标题1样忎】电工技术基础与技能 
【标题2样式】安全用电常识 
【标题3样式】触电的种类与形式

❌ 避免做法: 
1 电工技术基础与技能
  1.1 安全用电常识  
    1.1.1 触电的种类与形式
```

### Q: 认知维度标注不准确？
**A**: 在文档中使用明确的动词：
- 理解级: "理解..." "阐明..." "区别..."
- 应用级: "掌握..." "会使用..." "能够操作..."

### Q: 如何确保输出兼容图谱导入？
**A**: 验证以下几点：
1. 每行只有一个知识点 (A-G列只有一个非空)
2. H-I-J列关系使用英文分号";"分隔
3. L列(M列)只填入单项选择
4. 保留所有模板列(A-O)

## 6. 技巧和最佳实践

### 6.1 输出验证技巧
抽取完成后，验证三个关键点：

1. **树结构验证**: 检查 A-G 列是否每行只有一个知识点
```shell
# 统计每行列数 (应为1)
awk -F, 'NR>=2 {sum=0; for(i=1;i<=7;i++) {if($i != "") sum++}; print sum}' output.csv
```

2. **关系格式验证**: 检查H/J列是否使用英文分号
3. **分类合规验证**: L/M列是否为预定义值

### 6.2 质量提升策略  
- **使用结构化文档**: Word标题样式比PDF扫描更精确
- **嵌入提示词**: 在文档中明确标示认知层级
- **预定义模板**: 创建固定A1单元格的Excel模版，便于重复使用

## 7. 输出后续处理

### 7.1 导入知识图谱工具 (示例)
**导入 Neo4j**:
```cypher
LOAD CSV WITH HEADERS FROM 'file:///output.csv' AS row
WITH row WHERE row.`一级知识点` IS NOT NULL
CREATE (n:KnowledgeNode { 
  name: row.`一级知识点`,
  level: 1,
  cognitive: row.认知维度,
  category: row.分类
});
```

**导入 Jena Fuseki**:
- 将 CSV 转换为 RDF 格式后上传到三元组存储
