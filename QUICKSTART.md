# 快速入门：文档知识图谱抽取技能

## 🧠 技能核心能力

文档知识图谱抽取技能是一个专门从 **Word/PDF 教育文档** 自动抽取出 **层次化知识点** 的智能工具，能自动创建 **6+级树状结构** 并识别知识点间的语义关系，生成可用于知识图谱系统（Neo4j/Jena）导入的结构化数据。

### 技能价值
- **自动化**: 繁琐的文档结构化解析任务 → 一键生成
- **结构化**: 非结构化文本 → 知识图谱标准格式
- **关系化**: 静态知识点 → 带关系的知识网络
- **标准化**: 自定义格式 → 统一导入规范

---

## 🏁 三步上手

### 步骤1：准备两个文件

| 文件类型 | 要求 | 示例 |
|----------|------|------|
| **源文档** | `docx` 或 `pdf` 格式 <br>含清晰层级结构 | `05电子信息类.docx` <br> `课程标准.pdf` |
| **模板文件** | `xlsx` 格式 <br> A1单元格含格式规范 | `template.xlsx` |

### 步骤2：运行抽取命令

```bash
# 基础示例 (在技能目录下运行)
python scripts/extract_knowledge_graph.py \
  --source /path/to/your_document.docx \
  --template assets/template.xlsx \
  --output result.xlsx
```

### 步骤3：检查输出结果

- ✅ `result.xlsx` - 树状结构（A-G列每行一知识点）
- ✅ `H-I-J列` - 前置/后置/关联关系
- ✅ `L-M列` - 认知维度/知识分类
- ✅ 可直接导入知识图谱系统

## 👀 输出格式说明

### A-O列结构（示例片段）

| A | B | C | D | E | H | I | J | K | L | M |
|--|--|--|--|--|--|--|--|--|--|--|
| 电子信息考试 | 专业知识 | 电工技术 | 安全用电 | 触电形式 | 电压/电流概念 | 电位/电功率 | 安全措施... | 考点;重点 | 理解 | 概念性 |
| | | | | 测量方法 | 安全用电常识 | | | 难点;考点 | 应用 | 程序性 |

**核心约束**(必需)：
- ✅ **树状**：每行只能有1个知识点 (A-G列区间仅1列非空)
- ✅ **关系**：H-I-J用英文分号";"分隔多个知识点
- ✅ **单选**：L列(M列)只填单个认知维度(分类)
- ✅ **完整**：不能删除任一列(A-O列全保留)

## 🔍 关键使用技巧

### 技巧1：文档优化准备
```
✅ 推荐: 使用Word标题样式
【标题1】课程名称
  【标题2】模块名称  
    【标题3】章节名称
      【标题4】知识点组
        【标题5】具体知识点

❌ 避免: 纯数字编号
1. 课程名称
  1.1 模块名称
    1.1.1 章节名称
```

### 技巧2：认知层级提示词
在文档中使用明确词汇指导自动标注：
- **记忆层**: "了解" "知道" "说出" → `L列=记忆`
- **理解层**: "理解" "掌握" "说明" → `L列=理解`  
- **应用层**: "会使用" "能够操作" → `L列=应用`
- **分析层**: "分析" "比较" "辨析" → `L列=分析`

### 技巧3：关系识别提示
使用标识词语提示知识点间逻辑关系：
- **前置关系**: "基于...学习" "需先了解" → `H列(前置)`
- **关联关系**: "相关内容" "对比概念" → `J列(关联)`

## ⚠️ 常见错误回避

### 错误1：树状结构违例
```
❌ 错误: ['课程', '模块', '章节', '', '', '', '']  # 一行多节点
✅ 正确: ['课程', '', '', '', '', '', '']  # 每个A-G列仅1非空
         ['', '模块', '', '', '', '', '']  # 每行单一节点  
         ['', '', '章节', '', '', '', '']
```

### 错误2：层级跳跃
```
❌ 错误: ['','', '章节', '', '知识点', '', '']  # 跳过D列(四级)
✅ 正确: ['','', '章节', '主题', '知识点', '', '']  # 层级连续
```

### 错误3：关系混用
```
❌ 错误: A的前置是B，A的关联也是B (同一关系2种类型)
✅ 正确: A的前置是B，A的关联是C (区分关系类型)
```
