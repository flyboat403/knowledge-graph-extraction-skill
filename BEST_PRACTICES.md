# 技能使用最佳实践指南

## 核心工作流程

```
输入文档 (PDF/DOCX) → [技能处理] → 输出 (Excel/CSV)
```

**步骤分解：**

1. **源文档解析** - 从 docx/pdf 提取文本内容
2. **结构化分析** - 识别文档层次和内容主题
3. **知识点抽取** - 提取层次化知识节点（7级以上）
4. **关系识别** - 生成前置/后置/关联关系
5. **属性标注** - 标注认知维度和知识分类
6. **格式输出** - 生成树状Excel文件

---

## 输入文档优化建议

### 结构化文档准备

**最佳文档结构：**
```
一级标题: 课程名称/学科名称
  二级标题: 模块名称/单元名称  
  三级标题: 章节名称/部分  
    四级标题: 主题名称
      五级标题: 知识点标题
        六级标题: 细化知识点
```

### 关键词提示策略

在文档中使用明确的认知层级提示词:

| 认知层级 | 关键提示词 |
|----------|------------|
| **记忆** | "了解" "知道" "说出" "认识" "识别" "知道定义" |
| **理解** | "理解" "说明" "解释" "阐述" "区分" "归纳" "比较" |
| **应用** | "掌握" "会使用" "能够操作" "解决" "应用" "实施" |
| **分析** | "分析" "拆解" "剖析" "辨别" "分解" "比较" |
| **评价** | "评价" "判断" "选择" "评估" "论证" "验证" |
| **创造** | "设计" "创新" "规划" "构建" "创建" "改进" |

### 关系提示词

使用明确术语标示知识关系：

**前置关系:** "基于" "依赖" "以...为基础" "先决条件" "基础要求"  
**关联关系:** "相关" "联系" "类似" "相比" "与之比较" "同时"

| 列 | 教育含义 | 示例 |
|---|----------|------|
| A | 主题域 | "电子信息类技能考试" |
| B | 知识模块 | "专业知识(应知)" |
| C | 主题分类 | "电工技术基础与技能" |
| D | 领域子分类 | "安全用电常识" |
| E | 具体知识点 | "触电种类与形式" |
| F | 知识子项 | "接触触电" |
| G | 更细知识点 | "单相触电" |

### 认知层级判断指南

**判断树：**
```
文本中有"设计" → 创造
↓
文本中有"评价/选择/判断" → 评价  
↓
文本中有"分析/比较/分解" → 分析
↓
文本中有"应用/使用/操作" → 应用
↓
文本中有"理解/说明/解释" → 理解
↓
其他情况 → 记忆
```

### 知识分类规则

- **事实性**: 独立、具体的事实信息 (如："电阻单位为Ω")
- **概念性**: 概念、原理、定律之间关系 (如："欧姆定律表达式")  
- **程序性**: 如何做事情的方法步骤 (如："测量电流的方法")
- **元认知**: 关于认知的认知 (如："学习策略选择")

## 常见问题与解决

### Q: 为什么抽取的层级不够深？
**A**: 检查文档结构是否完整，确保使用层级标题样式而非仅靠序号

### Q: 认知维度判断不准确？
**A**: 在文档中使用明确动词，如"学生应掌握..."(→应用)、"学生应了解..."(→记忆)

### Q: 关系识别不准？
**A**: 文档中使用明确术语标示关系，如"基础概念是高级概念的前驱"、"A与B类似"

### Q: 输出格式错误？
**A**: 验证模板文件A1单元格是否包含格式要求

## 性能优化建议

### 大文档处理
- 分割长文档为多个子文档分别处理
- 一次处理不超过50页PDF或100页Word文档

### 提高准确率
1. 使用结构化样式（标题格式而非手动序号）
2. 包含明确的认知提示词
3. 标示知识点间的逻辑关系用语
4. 预处理文档（清除杂乱格式）

## 兼容性说明

### 导入工具
- **Neo4j**: 直接使用 CSV 文件
- **Jena**: 用 Excel 文件转换为 RDF  
- **图数据库**: 按 H-I-J 列构建节点关系

### 输出格式
- Excel: 直接打开编辑
- CSV: 可导入数据库分析
- 严格按照 A-O 列要求
