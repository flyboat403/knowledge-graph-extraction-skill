# 🧠 文档知识图谱抽取技能 - 速查卡片

## 📋 一行命令开始

```bash
python ~/.agents/skills/knowledge-graph-extractor/scripts/extract_knowledge_graph.py \
  --source document.docx \
  --template ~/.agents/skills/knowledge-graph-extractor/assets/template.xlsx \
  --output result.xlsx
```

---

## 🎯 三大核心功能

| 功能 | 输出示例 | 用途 |
|------|----------|------|
| **树状层级** | A列: 一级节点, B列: 二级节点... | 可视化知识层级 |
| **关系生成** | H列: 前置→I列: 后置→J列:关联 | 构建知识依赖网络 |
| **认知标注** | L列: 应用级别, M列: 概念性 | 教学目标层级 |

---

## 🔄 关键输入输出格式

### 输入规范  
- **源文档**: `.docx` 或 `.pdf` 文件  
- **模板文件**: `.xlsx` 文件，A1含格式约束
- **文档结构**: 使用 Word 标题样式（非手工编号）

### 输出列结构 (A-O列)

| 列 | 含义 | 约束 | 示例 |
|----|------|------|------|
| **A-G** | 知识点层级 | 每行仅1个非空 | `['课程', '', '模块', '', '知识点', '', '']` |
| **H** | 前置知识点 | 英文分号分隔 | `电阻概念;电流概念` |
| **I** | 后置知识点 | 自动生成 | `电位计算;功率测量` |
| **J** | 关联知识点 | 英文分号分隔 | `电压概念;电场概念` |
| **K** | 标签 | 重点/难点/考点 | `重点;考点` |
| **L** | 认知维度 | 单选值 | `记忆/理解/应用...` |
| **M** | 知识分类 | 単选值 | `事实性/概念性/程序性/元认知` |

### 🌳 层级含义示例  

```
A列: 电子信息类职业技能考试
 B列: 专业知识(应知)  
  C列: 电工技术基础与技能
   D列: 安全用电常识
    E列: 触电种类与形式 (前置: 电流概念;电压概念 | 关联: 安全措施)
```

## 🔍 认知层级判断（布鲁姆分类法）  

**动词优先级**:
```
[设计/创造/构建] → 🎨 创造
        ↓
[評价/选择/判断] → ⚖️ 评价  
        ↓
[分析/比较/辨析] → 🧩 分析  
        ↓
[应用/操作/使用] → 🔧 应用  ← 重点标注层级
        ↓
[理解/解释/阐述] → 💡 理解
        ↓
[了解/记住/知道] → 🧠 记忆
```

## 🚫 绝对避免的操作

### ❌ 格式违规

```
❌ 一行多知识点（破坏树状结构）
['电路基础', '安全用电', '触电种类', '', '', '', '']  # 错误！

✅ 正确做法（每行一个）:
['电路基础', '', '', '', '', '', '']
['', '', '安全用电', '', '', '', '']  
['', '', '', '', '触电种类', '', '']
```

### ❌ 关系违规

```  
❌ 使用中文分号/错误格式
H列: "知识点A；知识点B；知识点C"  # 错误！

✅ 用英文半角分号:
H列: "知识点A;知识点B;知识点C"   # 正确！
```

### ❌ 多值填入单选列

```
❌ 错误（L列/M列多值）
L列: "记忆;理解"  # 认知维度多选是错误
M列: "概念性;程序性"  # 分类多选是错误

✅ 正确（单选）
L列: "应用"  
M列: "程序性"
```

## ⚡ 工作流程

### 1. 输入预处理阶段 
- **文档检查**: 确保 Word/PDF 有清晰层级
- **模板读取**: 解析 xlsx 模板 A1 列格式规则
- **结构识别**: 自动识别标题等级/内容分区

### 2. 抽取处理阶段
```
源文档 → [分层解析] → [认知识别] → [关系抽取] → 输出格式化
```

### 3. 后处理验证
- **树结构验证**: 每行1节点 (A-G列) 
- **约束检查**: 关系分隔符格式, 认知维度单选
- **完整性验证**: 不删列，格式合规

## 🔧 调试指令

### 验证约束条件

```bash  
# 检查A-G列树结构
awk -F',' '{count=0; for(i1;i<=7;i++) if($i!= "") count++; if(count != 1) print "行"NRR": 非法格式, " count " 节点"}' output.csv

# 检查中文分号
grep -n '[；，]' .*.csv ..xlsx 2>/dev/null | grep -v ".*.md"

# 检认知维度
awk -F',' 'NR>1 {if(!/^(记忆|理解|应用|分析|评)*/ && $12 !~ /^[[:space:]]*$/ ) print "L列格式错误@行" NR ": " $12}' output.csv
```

### 文件准备检查清单

- [ ] Word标题样式 > 手编号
- [ ] 认知标识动词存在 ("掌握"/"理解"/"会")  
- [ ] 文档层级深度 ≥ 6级
- [ ] 模板 A1 有格式声明
- [ ] xlsx文件非损坏状态

## 💡 常见问答

**Q: 如何设置前置/后置/关联关系？**  
A: 系统自动识别，但可使用文档内语义词加强识别："基于...学习", "先...再..."

**Q: 认知层级判断准确率如何提升？**  
A: 在原文档中使用标准动词，如: "要掌握...", "需要理解...", "能够应用..."

**Q: 抽取结果如何导入Neo4j？**  
A: 使用H/I/J列构建关系，A-G列构建节点层次，L/M列做节点属性
