# 知识节点抽取 Prompt 模板

## 核心抽取 Prompt

```
你是一个知识图谱构建专家。请从以下文档中抽取知识节点和关系。

## 输入文档
{document_content}

## 输出格式要求

返回 JSON 数组，每个元素代表一个知识节点：

```json
[
  {
    "name": "知识点名称",
    "level": 4,
    "node_type": "知识点",
    "category": "概念性",
    "pre_requisites": ["前置知识点1"],
    "related": ["关联知识点1"],
    "tags": "重点;考点",
    "objective": "教学目标描述",
    "description": "知识点简要说明"
  }
]
```

## ⚠️ 关键：节点类型与知识分类约束

**node_type 字段决定 category 字段的填写规则**：

| node_type | category字段 | 适用层级 |
|-----------|-------------|----------|
| "分类" | **必须留空或不填** | Level 1-3 为主 |
| "知识点" | 填写分类值 | Level 4-7 为主 |

**判断规则（自动推断）**：
- Level 1-3 → node_type="分类"（课程、模块、章节等结构容器）
- Level 4-7 → node_type="知识点"（主题、具体知识点等学习内容）

**示例对比**：

```json
// 分类节点（Level 1-3）- category必须为空
{
  "name": "电工技术基础与技能",
  "level": 1,
  "node_type": "分类",
  "category": "",  // ← 分类节点必须留空
  "objective": "",
  "description": "电工技术课程"
}

// 知识点节点（Level 4-7）- category填写分类值
{
  "name": "欧姆定律",
  "level": 5,
  "node_type": "知识点",
  "category": "概念性",  // ← 知识点节点填写分类
  "objective": "能够理解欧姆定律的物理意义",
  "description": "电压、电流、电阻的关系定律"
}
```

## ⚠️ 关键：节点输出顺序要求

**必须按层级遍历顺序（深度优先）输出节点**：

1. 父节点必须在其所有子节点之前
2. 同一层级的兄弟节点按文档中的出现顺序排列
3. 一个分支的所有节点输出完毕后，再输出下一个分支

**正确示例**：
```
顺序: 节点类型(A列) + 课程(B列) → 模块(C列) → 章节(D列) → 章节的子知识点 → 下一个章节 → ...

例：
1. 电子信息类职业技能考试 (level 1, node_type="分类")
2.   专业知识(应知) (level 2, node_type="分类")
3.     电工技术基础与技能 (level 3, node_type="分类")
4.       安全用电常识 (level 4, node_type="知识点")
5.         触电的种类和形式 (level 5, node_type="知识点")
6.         触电预防措施 (level 5, node_type="知识点")
7.       电路基础 (level 4, node_type="知识点")
8.         电阻 (level 5, node_type="知识点")
9.         欧姆定律 (level 5, node_type="知识点")
10.    电子技术基础与技能 (level 3, node_type="分类")
11.      ...
12.  技能操作(应会) (level 2, node_type="分类")
13.    ...
```

**错误示例（禁止）**：
```
❌ 按层级排序输出（所有level 1 → 所有level 2 → ...）
❌ 子节点出现在父节点之前
❌ 不同分支的节点穿插输出
```

## 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 知识点名称（简洁准确，不超过30字） |
| level | int | 是 | 层级 1-7（1=课程级，7=原子知识点） |
| node_type | string | 是 | **新增**：节点类型，"分类" 或 "知识点" |
| category | string | **条件必填** | 知识分类：事实性/概念性/程序性/元认知（仅 node_type="知识点" 时填写） |
| pre_requisites | array | 否 | 前置知识点名称列表（学习依赖，非层级关系） |
| related | array | 否 | 关联知识点名称列表（相关但可独立学习） |
| tags | string | 否 | 标签，多个用英文分号分隔 |
| objective | string | **条件建议** | 教学目标（node_type="知识点" 时建议填写）。描述"学完后能做什么" |
| description | string | 否 | 知识点简要说明（不超过100字） |

## ⚠️ 教学目标（objective）生成规则

### 必填条件

| 层级 | node_type | 是否需要教学目标 | 原因 |
|------|-----------|-----------------|------|
| Level 1-3 | 分类 | ❌ 不需要 | 这些是结构容器，不是具体学习内容 |
| Level 4-7 | 知识点 | ✅ **建议填写** | 这些是学生实际学习的内容，应有明确目标 |

### 教学目标生成原则

**好的教学目标应描述：**
1. **学生学完后能做什么** - 使用行为动词（能够、会、掌握、理解）
2. **具体可衡量** - 避免模糊描述
3. **与知识点匹配** - 目标应该与知识点内容直接相关

### 教学目标句式模板

| 句式类型 | 推荐句式 | 示例 |
|----------|---------|------|
| 描述性 | 能够识别/说出/列举... | "能够说出电阻的单位及换算关系" |
| 理解性 | 能够解释/说明/理解... | "能够解释欧姆定律的物理意义" |
| 应用性 | 能够使用/操作/完成... | "能够使用万用表测量电阻值" |
| 分析性 | 能够分析/比较/区分... | "能够分析串联与并联电路的区别" |
| 评价性 | 能够评价/选择/判断... | "能够选择合适的电阻类型用于特定电路" |
| 创造性 | 能够设计/创建/构建... | "能够设计符合要求的简单电路" |

### 教学目标示例

**Level 4 (主题级)：**
```json
{
  "name": "幻灯片母版设计",
  "level": 4,
  "node_type": "知识点",
  "objective": "能够理解母版的概念和作用，掌握母版的基本操作方法"
}
```

**Level 5 (知识点级)：**
```json
{
  "name": "插入母版版式",
  "level": 5,
  "node_type": "知识点",
  "objective": "能够根据设计需求，在母版中插入新的自定义版式"
}
```

**Level 6 (细分级)：**
```json
{
  "name": "绝对引用",
  "level": 6,
  "node_type": "知识点",
  "objective": "理解绝对引用的概念，能够在公式中正确使用$符号锁定单元格"
}
```

### 错误示例（禁止）

```json
// ❌ 太模糊
"objective": "学会这个知识"

// ❌ 与知识点无关
"objective": "培养学生的创新能力"

// ❌ Level 1-3 分类节点不应该有教学目标
{ "name": "演示文稿", "level": 1, "node_type": "分类", "objective": "..." }  // 课程名不需要目标
```

## 层级分配规则

| level | Excel列位置 | 典型内容 | 示例 | node_type |
|-------|-------------|----------|------|-----------|
| 1 | B列 | 课程/学科名称 | 电子信息类职业技能考试 | 分类 |
| 2 | C列 | 模块/单元 | 专业知识(应知) | 分类 |
| 3 | D列 | 章节/部分 | 电工技术基础与技能 | 分类 |
| 4 | E列 | 知识主题 | 安全用电常识 | 知识点 |
| 5 | F列 | 具体知识点 | 触电的种类和形式 | 知识点 |
| 6 | G列 | 知识点细分 | 接触触电的类型 | 知识点 |
| 7 | H列 | 原子内容 | 单相触电的具体特征 | 知识点 |

## 前置关系 vs 层级关系（重要区分）

| 概念 | 定义 | JSON 字段 | 示例 |
|------|------|-----------|------|
| **层级关系** | 知识点的归属关系 | 由输出顺序+level确定 | "电阻" 属于 "电路基础" |
| **前置关系** | 学习依赖关系 | pre_requisites | 学 "欧姆定律" 前要先学 "电阻" |

**注意**：
- 层级关系决定节点在树中的位置（通过输出顺序表达）
- 前置关系决定学习路径（通过 pre_requisites 字段表达）
- 两者可能不同：例如 "欧姆定律" 的父节点是 "电路基础"，但前置关系可能是 "电阻"

## 抽取原则

### 0. ⚠️ 排除内容（禁止抽取）

以下内容属于**教学管理信息**，不属于学科知识，**禁止抽取**：

| 排除类型 | 示例 |
|----------|------|
| 课程管理信息 | 课程性质、课程任务、课程地位、教学学时、学分、证书信息 |
| 教学目标描述 | 课程核心素养、思政目标、素质目标 |
| 教学安排信息 | 教学策略、教学方法、教学条件、教学团队要求、教学要求 |
| 评价标准信息 | 评价原则、评价方式、毕业条件|
| 教材编写要求 | 内容选择、呈现形式、体例结构 |
| 课程资源信息 | 文本资源、数字化资源、设施设备 |
| 校企合作信息 | 企业专家参与、实训基地建设 |

**判断测试**：问自己"学生需要学习这个知识吗？"
- 如果答案是"这是教学安排，不是学习内容" → 不抽取
- 如果答案是"学生需要掌握这个技能/概念" → 抽取

**正确示例**：
- "Word文档的段落格式设置方法" ✅ 学科知识
- "Excel中VLOOKUP函数的使用方法" ✅ 学科知识
- "PPT幻灯片母版的设计原则" ✅ 学科知识

**错误示例（禁止抽取）**：
- "本课程计划教学学时为216学时" ❌ 教学安排
- "教学团队规模：专职教师2名" ❌ 教学管理
- "学业质量分为水平一和水平二" ❌ 评价标准
- "采用任务教学法进行教学" ❌ 教学方法
- "课程培养信息素养目标" ❌ 教学目标

### 1. 知识节点识别

**应该抽取**：学科/专业知识内容
- 具体的概念、定义、原理
- 操作方法、步骤、技巧
- 功能特性、参数配置
- 问题解决策略

**不应抽取的**：
- 考试说明、评分标准等行政内容
- "本章主要内容"等导读性文字
- 过于细碎无法独立存在的内容
- **教学管理信息**（见上方排除列表）

### 2. 层级判断

判断层级的三个问题：
- Q1: 这个知识点属于哪个上级模块？（确定父节点）
- Q2: 这个知识点包含哪些下级内容？（确定子节点）
- Q3: 这个知识点与哪些同级概念并列？（确定兄弟节点）

### ⚠️ 关键：节点类型判断优先规则

**判断node_type时，应优先考虑是否有下级知识点**：

| 条件 | node_type | 说明 |
|------|-----------|------|
| 有下级知识点（children） | **分类** | 作为结构容器，不应有知识点分类 |
| 无下级知识点（叶子节点） | **知识点** | 可以有知识点分类（事实性/概念性/程序性/元认知） |

**层级辅助判断**（当无法确定是否有下级时）：
- Level 1-3 → 通常是分类（课程、模块、章节）
- Level 4-7 → 通常是知识点（主题、具体内容）

**重要**：如果Level 4节点包含Level 5子节点，这个Level 4节点必须是"分类"！

### 3. 节点类型判断（新增）

**根据层级自动判断 node_type**：
- Level 1-3 → node_type="分类"
- Level 4-7 → node_type="知识点"

**根据 node_type 决定 category 字段**：
- node_type="分类" → category=""（必须留空）
- node_type="知识点" → category="事实性/概念性/程序性/元认知"

### 4. 前置关系识别

识别学习依赖，以下情况应建立前置关系：

| 模式 | 示例 |
|------|------|
| 基础概念 → 高级应用 | 电阻 → 欧姆定律 |
| 理论原理 → 实践操作 | 电路原理 → 电路安装 |
| 工具使用 → 应用场景 | 万用表使用 → 电流测量 |

**⚠️ 关系引用必须精确匹配节点完整名称**：
- pre_requisites 中的节点名必须与 JSON 中已定义的节点 `name` 字段**完全一致**
- 禁止使用缩写、简化名称或部分名称
- 建议在生成关系前先检查目标节点是否已存在于 JSON 数组中

**正确示例**：
```json
// 节点定义
{ "name": "半身裙结构图的绘制与工业样板的制作", "level": 4, ... }

// 前置关系引用（必须使用完整名称）
{ "name": "裤装结构图的绘制与工业样板的制作", "pre_requisites": ["半身裙结构图的绘制与工业样板的制作"], ... }
```

**错误示例（禁止）**：
```json
// ❌ 使用缩写名称（缺少"与工业样板的制作"后缀）
{ "pre_requisites": ["半身裙结构图的绘制"] }  // 引用不存在！

// ❌ 使用简化名称
{ "pre_requisites": ["人体测量方法"] }  // 实际节点名是"人体测量方法及规范"
```

### 5. 关联关系识别

识别相关但不构成依赖的知识点：

| 模式 | 示例 |
|------|------|
| 同类并列 | 电阻 ↔ 电容 ↔ 电感 |
| 对比概念 | 直流电路 ↔ 交流电路 |
| 相似原理 | 欧姆定律 ↔ 基尔霍夫定律 |

**⚠️ 关系引用必须精确匹配节点完整名称**：
- related 中的节点名必须与 JSON 中已定义的节点 `name` 字段**完全一致**
- 禁止使用缩写、简化名称或部分名称
- 建议在生成关系前先检查目标节点是否已存在于 JSON 数组中

**正确示例**：
```json
// 节点定义
{ "name": "服装效果图绘制", "level": 5, ... }
{ "name": "服装平面款式图绘制", "level": 5, ... }

// 关联关系引用（必须使用完整名称）
{ "name": "服装效果图绘制", "related": ["服装平面款式图绘制"], ... }
```

**错误示例（禁止）**：
```json
// ❌ 使用缩写名称
{ "related": ["款式图绘制"] }  // 引用不存在！

// ❌ 名称不匹配
{ "related": ["熨烫操作方法"] }  // 实际节点名是"熨烫基本知识与操作方法"
```

### 6. 知识分类判断

| 分类 | 说明 | 示例 |
|------|------|------|
| 事实性 | 具体事实、数据 | 电压单位是伏特 |
| 概念性 | 概念、原理、定义 | 欧姆定律的定义 |
| 程序性 | 操作步骤、方法 | 万用表测量电阻的方法 |
| 元认知 | 学习策略、方法选择 | 如何选择合适的解题方法 |

## 输出检查清单

抽取完成后，请检查：

- [ ] **节点顺序**：父节点在子节点之前，按深度优先顺序
- [ ] **节点类型正确**：Level 1-3="分类"，Level 4-7="知识点"
- [ ] **知识分类约束**：分类节点 category为空，知识点节点填写分类值
- [ ] **教学目标完整性**：知识点的教学目标描述具体可衡量
- [ ] **教学目标质量**：教学目标使用行为动词，描述具体可衡量的学习成果
- [ ] **关系引用完整性**：pre_requisites 和 related 中的节点名必须与 JSON 中已定义的节点 name 字段**完全一致**
- [ ] **关系引用无缩写**：禁止使用简化名称或部分名称作为关系引用
- [ ] 每个知识点名称简洁准确（不超过30字）
- [ ] 层级正确反映知识的从属关系
- [ ] 前置关系符合学习逻辑
- [ ] 已过滤非知识内容（考试说明、行政信息）

**⚠️ 关系引用完整性验证方法**：
在输出 JSON 前，请逐一检查每个节点的 pre_requisites 和 related 字段：
1. 对于每个引用的节点名，在 JSON 数组中搜索是否存在完全匹配的 `name` 字段
2. 如果找不到匹配节点，说明引用错误，需要修正或删除该引用
3. 常见错误：节点名含后缀（如"与工业样板的制作"），但引用时遗漏了后缀

请开始抽取：
```
