## 列结构定义

| 列 | 字段名 | 用途 | 约束 |
|----|--------|------|------|
| A | 一级知识点 | 最高层级 | 课程/学科名称 |
| B | 二级知识点 | 模块层级 | 模块/单元名称 |
| C | 三级知识点 | 章节层级 | 章节/部分名称 |
| D | 四级知识点 | 主题层级 | 主题/知识点组 |
| E | 五级知识点 | 具体知识点 | 最细粒度知识点 |
| F | 六级知识点 | 扩展层级 | 可选 |
| G | 七级知识点 | 扩展层级 | 可选 |
| H | 前置知识点 | 学习前置依赖 | 多个用 `;` 分隔 |
| I | 后置知识点 | 学习后续内容 | 多个用 `;` 分隔 |
| J | 关联知识点 | 相关知识点 | 多个用 `;` 分隔 |
| K | 标签 | 重点/难点标记 | 多个用 `;` 分隔 |
| L | 认知维度 | 学习目标层次 | 单选 |
| M | 分类 | 知识类型 | 单选 |
| N | 教学目标 | 学习目标描述 | 文本 |
| O | 知识点说明 | 详细说明 | 文本 |

## 层级关系约束

### 核心规则：树状结构

**关键约束**：每一行只能填写一个知识点（A-G列中只有一个非空单元格）

知识点所在的列位置表示其层级：
- A列 = 一级（根节点）
- B列 = 二级（A的子节点）
- C列 = 三级（B的子节点）
- D列 = 四级（C的子节点）
- E列 = 五级（D的子节点）
- F列 = 六级（E的子节点）
- G列 = 七级（F的子节点）

### 层级连接约束

1. **连续性要求**：知识点的层级必须向上连续
   - ✅ 正确：从A→B→C→D逐级展开
   - ❌ 错误：从A→C跳跃（跳过B），除非B层内容确实不存在

2. **单向性要求**：每行只有一个非空知识点
   - ❌ 错误：`['电子产品', '专业知识', '电工技术', '安全用电', '触电预防', '', '']`（一行6个知识点）
   - ✅ 正确：`['电子产品', '', '', '', '', '', '']` （一行1个知识点）

### 正确示例

```
行号    A列            B列              C列            D列            E列              F列
3       电子信息技术  
4                      专业知识         电工技术                        
5                                     安全用电          触电种类
6                                     安全员            证书
7                      专业知识         电子技术
8                                     半导体            PN结特性
```

## 知识点关系约束

### H列：前置知识点（Pre-requisite Knowledge Points）

学习当前知识点**之前**必须先掌握的知识点。

**识别规则：**

1. **基础知识 → 高级知识**  
   - 示例：`电阻` → `欧姆定律`
   - 说明：理解欧姆定律**前**需要掌握电阻概念

2. **工具使用 → 应用操作**
   - 示例：`万用表的使用` → `测量电流`
   - 说明：测量电流**前**需要学会使用万用表

3. **理论知识 → 实践技能**  
   - 示例：`电路原理` → `电路安装`
   - 说明：电路安装**前**需要理解电路原理

4. **序列依赖**
   - 示例：`第1章` → `第2章`
   - 说明：后续章节**前**通常需要掌握前置章节

**生成策略：**
- 在文档中根据 `然后...`, `接下来...`, `先...再...` 等表达识别依赖关系
- 根据概念的复杂性确定难易顺序
- 基础→进阶, 简单→复杂, 一般→特殊

### I列：后置知识点（Post-requisite Knowledge Points）

学习当前知识点**之后**可以继续学习的知识点。

**生成规则：**
- 自动从前置关系中推导：
  - 如果 "A 的前置是 B"，则 "B 的后置是 A"
  - 确保关系的**双向性**和**一致性**

**示例：**
- A列知识点："电阻"
- H列关系："电流;电压"
- 则以下知识点的 I列包含"电阻"：
  - "电流" → I列追加："电阻"
  - "电压" → I列追加："电阻"

### J列：关联知识点（Related Knowledge Points）

与当前知识点**相关但不构成前后置关系**的知识点。

**识别规则：**

1. **并列主题**
   - 示例：`直流电路` ↔ `交流电路` 
   - 说明：主题相同不同类别的知识点

2. **相似概念对别**
   - 示例：`电压` ↔ `电流` 
   - 说明：具有相似地位、作用对别的概念

3. **逻辑相关非依赖**
   - 示例：`功率因数` ↔ `效率计算`
   - 说明：在同一范围内讨论但相互独立的概念

4. **主题扩展**
   - 示例：`放大电路` ↔ `振荡电路`
   - 说明：都属于`电子技术`但功能不同的知识点

**关联识别策略：**
- 在同一章节或段落中频繁同时出现的概念
- "还有..."、"另外..."、"与此同时..."等表达中的知识点
- 同一类别下的不同选项（如：`电阻`、`电容`、`电感`）

### 关系约束

1. **唯一性**：任意两个知识点之间**只能存在一种关系类型**（前置、后置或关联）
   - ❌ 错误：对于"欧姆定律"，它既是A的前置也是B的后置还是C的关联
   - ✅ 正确：一个知识点关系类型单一

2. **容量限制**：关系总数上限 2000 个
   - 如果达到限制，新导入的关系将覆盖旧关系（优先保留更密切的关系）

3. **格式约束**：
   - 多个知识点用**英文分号 `;`** 分隔  
   - ❌ 错误：`知识点A；知识点B` (中文分号)
   - ✅ 正确：`知识点A;知识点B` (英文分号)

## 标签与分类约束

### K列：标签（Labels）

**固定标签**（根据认知重要性标记）：
| 标签名 | 适用条件 | 示例 |
|--------|----------|------|
| 重点 | 学科核心知识，考查频率高 | `电阻`、`欧姆定律` |
| 难点 | 学生学习困难大，理解程度低 | `交流电路分析`、`放大器失真` | 
| 考点 | 历年考题常出现的知识点 | `安全用电常识`、`三态门逻辑` |
| 课程思政 | 蕴含育人价值的教学内容 | `工程伦理`、`社会责任` |

**自定义标签**：可根据需要创建
- 形式：有意义的专业术语（如：`必考`、`易错`、`速记` 等）
- 注意：自定义前优先使用固定标签，以保持规范性

**多标签格式**：`重点;考点`、`难点;易错` 

**标签生成策略：**
- 优先判断文档中使用的重要程度标示词汇
- 根据考试大纲标注的权重/比例判断重要性
- 根据知识点在教学实践/考试中的出现频率判断

### L列：认知维度（Cognitive Level）

**定义标准**：基于布鲁姆教育目标分类法

| 维度 | 英文 | 说明 | 典型动词 | 识别关键词 | 示例 |
|------|------|------|----------|------------|------|
| 记憶 | Remember | 识别回忆信息 | 识认、回忆、复述 | "说出"、"记住"、"背诵" | 知道电压的单位是伏特 |
| 理解 | Understand | 解释说明含义 | 解释、阐述、总结 | "理解"、"明白"、"说明" | 理解欧姆定律的含义 |
| 应用 | Apply | 在新情景中使用 | 使用、应用、操作 | "掌握"、"能够"、"会" | 利用万用表测量电阻值 |
| 分析 | Analyze | 分解要素关系 | 比较、分解、辨析 | "分析"、"分析结构"、"对比" | 分析三极管放大电路的工作原理 |
| 评价 | Evaluate | 做出判断论证 | 判断、比较、评价 | "评价"、"选择"、"对比" | 评价不同电路设计方案的优劣 |
| 创造 | Create | 生成新的方案 | 设计、创造、制定 | "设计"、"规划"、"构建" | 设计一个稳压电源电路 |

**单选约束**：每行知识点只能选择一个认知维度。

### M列：知识点分类（Knowledge Category）

**定义标准**：基于知识性质对知识点进行分类

| 分类 | 说明 | 识别特征 | 典型动词 | 示例 |
|------|------|----------|----------|------|
| 亊実性 | 独立存在的基本要素 | 具体的名词、符号、日期、数值 | "说出"、"记住" | 交流电的频率是50Hz |
| 概念性 | 组织和结构良好的知识体系 | 原理、理论、定律、法则、模型 | "理解"、"解释" | 欧姆定律、PID控制器原理 |
| 程序性 | 如何完成某种活动的知识 | 步骤、流程、规则、策略、技巧 | "会"、"操作"、"实施" | 使用示波器观察波形的方法 |
| 元認知 | 关于认知的知识 | 学习策略、反思、监控、调节 | "计划"、"监控"、"评价" | 如何选择合适的测量仪器 |

**单选约束**：每行知识点只能选择一个知识分类。

## 正确与错误示例

### 完整行示例（正确格式）

**一级知识点示例 (A列):**
```
A: 电子信息类职业技能考试
B-G: 空白
H: 无 (作为根节点)
I: "专业知识(应知);技能操作(应会)"  
J: 无
K: "重点"
L: "理解"
M: "概念性"
N: "掌握电子信息类职业技能考试的基本框架"
O: "四川省普通高校招生职业技能考试大纲"
```

**二级知识点示例 (B列) - 与根节点有关系:**
```
A: ""
B: 专业知识(应知)
C-G: 空白
H: "电子信息类职业技能考试" (前置关系)
I: "电工技术基础与技能;电子技术基础与技能;单片机原理及应用" (后置关系)
J: "技能操作(应会)" (关联关系，应知与应会并列)
K: ""
L: "理解"
M: "概念性"  
N: "理解专业知识(应知)的考试要求"
O: "专业知识考试部分分值200分，约占总分的57%"
```

**四级知识点示例 (D列) - 完整关系与标签:**
```
A-C: 空白
D: 电动势和欧姆定律
E-G: 空白
H: "电阻;电流;电压" (前置: 这些是学习此知识点前应掌握的知识)
I: "电位;电能和电功率;基尔霍夫定律" (后置: 学完后可学习这些)
J: "电源内外特性" (关联: 相关但非直接依赖)  
K: "重点;考点"
L: "应用"
M: "概念性"
N: "重点掌握部分电路欧姆定律和全电路欧姆定律的应用"
O: "主要内容包括部分电路欧姆定律、全电路欧姆定律及其应用实例"
```

**五级知识点示例 (E列) - 复杂认知级别:**
```
A-D: 空白
E: 电位的含义与计算方法
F-G: 空白
H: "电压的基本概念" (前置关系)
I: "" (无后置，已到细节知识点)  
J: "电势;零电位点的选择" (关联知识点)
K: "考点"
L: "应用"
M: "概念性"
N: "掌握电路中各点电位的计算方法，并能灵活应用"
O: "涉及电位与电压关系、参考点选择对电位的影响等内容"
```

### 常见错误示例（应避免）

```
❌ 错误1：一行填写多个知识点（违反树状结构）
['电子产品', '专业知识', '电工技术', '', '', '', '']  # 一列中有3个知识点

✅ 正确：
['电子产品', '', '', '', '', '', '']    # 每行一个知识点
['', '专业知识', '', '', '', '', '']    # 作为子节点
['', '', '电工技术', '', '', '', '']    # 作为孙子节点

❌ 错误2：层级跳跃（非连续）
['', '', '', '', '电阻', '', '']    # 跳过了D列'电路基础'
['', '专业', '', '', '器件', '', '']  # 跳过了B→C→D列

✅ 正确：中间层级用空字符串保持连续性
['电子信息类职业技能考试', '', '', '', '', '', '']
['', '专业知识(应知)', '', '', '', '', '']
['', '', '电工技术基础与技能', '', '', '', '']  
['', '', '', '电路基础', '', '', '']
['', '', '', '', '电阻', '', '']

❌ 错误3：错误的认知维度拼写
L: "应用和理解"  # ❌ 多选
L: "applied"      # ❌ 英文

✅ 正确：
L: "应用"         # ✅ 单选

❌ 错误4：错误的关系分隔符
H: "电阻；电流；电压"  # ❌ 使用中文分号
J: "知识点1,知识点2"   # ❌ 使用逗号  

✅ 正确：
H: "电阻;电流;电压"   # ✅ 使用英文分号
```

### 约束检查清单

- [ ] ✅ 每行(A-G列)只填写一个知识点
- [ ] ✅ 遵循树状层级结构（无跳跃）
- [ ] ✅ H/I/J列使用英文分号 `;` 分隔知识点  
- [ ] ✅ L列认知维度：记忆/理解/应用/分析/评价/创造
- [ ] ✅ M列认知分类：亊実性/概念性/程序性/元認知
- [ ] ✅ K列使用固定标签或合理自定义标签
- [ ] ✅ 同两个知识点之间只有一种关系类型（前置/后置/关联）
- [ ] ✅ 保持模板列完整性和表头完整性
