## ⚠️ 列名固定约束（重要）

**Excel第二行的列名是固定标准，不可修改**：

列名由脚本硬编码生成，用于下游导入系统（Neo4j/Jena/超星平台）。列名不匹配会导致导入失败。

**Agent和LLM不得修改列名** - 列名校验会自动检查输出文件是否符合标准。

### 标准列名列表

| 列 | 固定列名 | 说明 |
|----|----------|------|
| A | 节点类型 | 固定 |
| B-H | 节点名称(共7列) | 固定，表示层级 |
| I | 前置节点 | 固定 |
| J | 后置节点 | 固定 |
| K | 关联节点 | 固定 |
| L | 标签 | 固定 |
| M | 知识点分类 | 固定 |
| N | 节点说明 | 固定 |
| O | 教学目标 | 固定 |

## 列结构定义

| 列 | 字段名 | 用途 | 约束 |
|----|--------|------|------|
| A | 节点类型 | 节点分类标识 | **必填**：填写"分类"或"知识点" |
| B | 节点名称 | 一级节点 | 课程/学科名称 |
| C | 节点名称 | 二级节点 | 模块/单元名称 |
| D | 节点名称 | 三级节点 | 章节/部分名称 |
| E | 节点名称 | 四级节点 | 主题/知识点组 |
| F | 节点名称 | 五级节点 | 具体知识点 |
| G | 节点名称 | 六级节点 | 扩展层级 |
| H | 节点名称 | 七级节点 | 扩展层级 |
| I | 前置节点 | 学习前置依赖 | 多个用 `;` 分隔 |
| J | 后置节点 | 学习后续内容 | 多个用 `;` 分隔 |
| K | 关联节点 | 相关知识点 | 多个用 `;` 分隔 |
| L | 标签 | 重点/难点标记 | 多个用 `;` 分隔 |
| M | 知识点分类 | 知识类型 | **分类节点必须留空** |
| N | 节点说明 | 详细说明 | 文本 |
| O | 教学目标 | 学习目标描述 | 文本 |

## 节点类型约束

### A列：节点类型

节点类型决定M列（知识点分类）的填写规则，这是**最关键的约束**。

| 节点类型 | A列值 | 适用层级 | 知识点分类(M列) | 其他属性 |
|----------|-------|----------|-----------------|----------|
| **分类** | `分类` | Level 1-3 为主 | **必须留空** | 可有层级、关系、标签 |
| **知识点** | `知识点` | Level 4-7 为主 | 填写分类值 | 可有完整属性 |

### 判断规则

**根据层级自动判断节点类型**：

| 层级 | 节点类型 | 原因 |
|------|----------|------|
| Level 1 (课程) | 分类 | 结构容器，非具体学习内容 |
| Level 2 (模块) | 分类 | 结构容器，非具体学习内容 |
| Level 3 (章节) | 分类 | 结构容器，非具体学习内容 |
| Level 4 (主题) | 知识点 | 学生实际学习的内容 |
| Level 5 (知识点) | 知识点 | 学生实际学习的内容 |
| Level 6-7 | 知识点 | 学生实际学习的具体内容 |

### ⚠️ 关键规则：分类节点M列必须留空

```
❌ 错误：分类节点填写了知识分类
A列="分类", M列="概念性"  # 禁止！

✅ 正确：分类节点M列为空
A列="分类", M列=""  # 必须！

✅ 正确：知识点节点填写知识分类
A列="知识点", M列="概念性"  # 允许
```

## 层级关系约束

### 核心规则：树状结构

**每行只能填写一个节点名称**（B-H列中只有一个非空单元格）

节点名称所在的列位置表示其层级：
- B列 = 一级（根节点）
- C列 = 二级（B的子节点）
- D列 = 三级（C的子节点）
- ... 以此类推到 H列 = 七级

### 正确示例

```
行号  A列(类型)  B列(一级)      C列(二级)        D列(三级)        E列(四级)    
3     分类       演示文稿
4     分类                      新建演示文稿
5     知识点                                    幻灯片大小
6     分类                      演示文稿设计
7     知识点                                    主题
8     知识点                                                    背景
9     分类                      增删幻灯片
10    知识点                                    新建幻灯片
```

### 错误示例（禁止）

```
❌ 错误：一行填写多个层级
['演示文稿', '新建演示文稿', '幻灯片大小', '', '', '', '']

✅ 正确：每行一个节点名称
['分类', '演示文稿', '', '', '', '', '', '']
['分类', '', '新建演示文稿', '', '', '', '', '']
['知识点', '', '', '幻灯片大小', '', '', '', '']
```

## ⚠️ 节点类型与子节点关系约束（重要）

**核心规则**：
- **知识点节点不能有子节点**（叶子节点）
- **分类节点可以有子节点**（结构容器）

**自动修正机制**：
若生成的JSON中"知识点"节点包含子节点，脚本会自动将其修正为"分类"节点，
并清空其"知识点分类"字段。

**修正示例**：
| 原JSON | 修正后 |
|--------|--------|
| node_type="知识点", children=[...] | node_type="分类", category="" |

## 知识点关系约束

### I列：前置节点

学习当前节点前需要先掌握的节点。

**识别规则：**
1. 基础概念 → 高级概念（如：电阻 → 欧姆定律）
2. 工具使用 → 应用操作（如：万用表使用 → 电流测量）
3. 理论知识 → 实践技能（如：电路原理 → 电路安装）

### J列：后置节点

学习当前节点后可以继续学习的节点。

**生成规则：**
- 自动由 I列 生成：如果 A的前置是B，则 B的后置是A
- 确保关系的双向性

### K列：关联节点

与当前节点相关但不构成前置/后置关系的节点。

**识别规则：**
1. 同一主题下的并列节点
2. 相似概念或对比概念
3. 不同章节但逻辑相关的节点

### 关系约束

1. **唯一性**：任意两个节点之间只能存在一种关系
2. **数量上限**：关系设置总数上限 2000 个
3. **分隔符**：多个节点用英文分号 `;` 分隔

## 标签与分类约束

### L列：标签

| 固定标签 | 说明 |
|----------|------|
| 重点 | 核心知识点 |
| 难点 | 学生普遍困难的知识点 |
| 考点 | 考试常考知识点 |
| 课程思政 | 思政教育相关知识点 |

可自定义标签，多个标签用 `;` 分隔。

### M列：知识点分类（单选）

| 分类 | 说明 | 示例 |
|------|------|------|
| 事实性 | 具体事实、数据 | 电压单位是伏特 |
| 概念性 | 概念、原理、定义 | 欧姆定律的定义 |
| 程序性 | 操作步骤、方法 | 万用表测量电阻的方法 |
| 元认知 | 学习策略、方法选择 | 如何选择合适的解题方法 |

**⚠️ 重要约束**：分类节点不支持填写知识点分类，M列必须为空。

## 正确与错误示例

### 完整行示例

**分类节点示例（Level 1）- 根节点:**
```
A: 分类
B: 演示文稿
C-H: (空)
I: (空)
J: (空)
K: (空)
L: (空)
M: (空)  ← 分类节点必须留空
N: (空)
O: (空)
```

**分类节点示例（Level 2）- 有后置关系:**
```
A: 分类
B: (空)
C: 新建演示文稿
D-H: (空)
I: (空)
J: 增删幻灯片;新建幻灯片;删除幻灯片
K: (空)
L: (空)
M: (空)  ← 分类节点必须留空
N: (空)
O: (空)
```

**知识点节点示例（Level 4）- 有完整属性:**
```
A: 知识点
B-D: (空)
E: 幻灯片背景
F-H: (空)
I: (空)
J: (空)
K: (空)
L: 应用;事实性
M: 事实性  ← 知识点节点可填写分类
N: 设置幻灯片背景的方法
O: 能够掌握设置幻灯片背景的操作方法
```

**知识点节点示例（Level 5）- 有前置关系:**
```
A: 知识点
B-E: (空)
F: 字号
G-H: (空)
I: (空)
J: 字体格式
K: (空)
L: 容易;应用
M: 事实性
N: 字号的设置方法
O: 能够正确设置文字字号
```

### Anti-Patterns（禁止）

| 错误 | 正确 | 说明 |
|------|------|------|
| A列="分类", M列="概念性" | A列="分类", M列="" | 分类节点M列必须留空 |
| 一行填写多个节点名称 | 每行只填一个节点名称 | 树状结构要求 |
| 使用中文分号 `；` | 使用英文分号 `;` | 解析器仅识别ASCII分号 |
| 知识点分类填多个值 | 知识点分类为单选 | M列只能填写一个分类 |
| 层级跳跃（B→D） | 层级连续（B→C→D） | 每个节点必须有完整父链 |
