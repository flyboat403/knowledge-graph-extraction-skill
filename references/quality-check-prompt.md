# 质量校验 Prompt 模板

## Agent 自检与修正 Prompt

```
你已完成知识节点抽取，生成了 JSON 文件。现在执行质量校验并修正问题。

## 当前 JSON 文件路径
{json_file_path}

## 校验清单（9项必检）

逐一检查以下内容，发现问题立即使用 Edit 工具修正：

### 1. 教学目标覆盖（关键）
- 检查：Level 4-7 的知识点节点是否都有 objective 字段
- 问题判定：level≥4 但 objective="" 或 objective="无"
- 修正方法：使用 Edit 工具，添加具体教学目标
- 目标格式："能够[行为动词]...[具体内容]"
- 示例修正：
  ```json
  // 修正前
  {"name": "欧姆定律", "level": 5, "objective": ""}
  // 修正后
  {"name": "欧姆定律", "level": 5, "objective": "能够理解欧姆定律的物理意义，掌握电压、电流、电阻的关系计算"}
  ```

### 2. 教学目标质量
- 检查：objective 是否使用行为动词、是否具体可衡量
- 问题判定：模糊描述如"学会这个"、"了解"、"掌握知识"
- 修正方法：改为具体行为目标
- 行为动词：能够说出/能够解释/能够操作/能够分析/能够设计/能够判断

### 3. 层级深度（导入要求）
- 检查：最高层级 level 是否 ≥ 6
- 问题判定：max(level) < 6
- 修正方法：在末级知识点下添加细分节点（level 6-7）
- 示例修正：在 level 5 的"电阻"下添加 level 6 的"电阻测量方法"

### 4. 层级跳跃
- 检查：是否存在断层（父→孙跳过子）
- 问题判定：连续节点中 level 从 1 直接到 3
- 修正方法：补充缺失的中间层级节点

### 5. 知识分类约束（关键）
- 检查：node_type="分类" 的节点，category 是否为空
- 问题判定：分类节点有 category 值
- 修正方法：使用 Edit 工具，清空 category 字段
- 示例修正：
  ```json
  // 修正前
  {"name": "电路基础", "node_type": "分类", "category": "概念性"}
  // 修正后
  {"name": "电路基础", "node_type": "分类", "category": ""}
  ```

### 6. 知识点分类覆盖
- 检查：Level 4-7 的知识点节点是否有 category
- 问题判定：node_type="知识点" 且 level≥4 但 category=""
- 修正方法：根据内容类型填写分类
- 分类值：事实性/概念性/程序性/元认知

### 7. 名称长度
- 检查：name 是否超过 30 字
- 问题判定：len(name) > 30
- 修正方法：缩短名称，保留核心含义

### 8. 关联关系覆盖
- 检查：知识点节点的 related 是否有值
- 问题判定：>50% 的知识点 related=[]
- 修正方法：补充同类/对比/相似原理的关联节点名称
- 示例：电阻 的 related 可填 ["电容", "电感"]

### 9. 节点顺序（深度优先）
- 检查：父节点是否在子节点之前
- 问题判定：JSON 数组中子节点排在父节点前面
- 修正方法：调整节点顺序，确保深度优先输出

### 10. 列名结构（格式校验）
- 检查：生成的Excel第二行列名是否与模板完全一致
- 问题判定：任意列名与标准不符
- 修正方法：⚠️ 列名由脚本硬编码，若出现差异需重新运行脚本生成
- 标准列名：A:节点类型, B-H:节点名称(7列), I:前置节点, J:后置节点, K:关联节点, L:标签, M:知识点分类, N:节点说明, O:教学目标

## 执行流程

1. 使用 Read 工具读取 JSON 文件完整内容
2. 遍历所有节点，逐一检查上述 9 项
3. 发现问题 → 立即使用 Edit 工具修正
4. 修正完成后重新读取并校验（循环）
5. 全部通过后输出校验报告

## 校验通过标准

| 检查项 | 通过标准 |
|--------|----------|
| 教学目标覆盖 | Level 4-7 节点 100% 有 objective |
| 教学目标质量 | 所有 objective 包含行为动词 |
| 层级深度 | max(level) ≥ 6 |
| 知识分类约束 | 分类节点 category="" |
| 知识点分类覆盖 | 知识点节点有 category 值 |
| 关联关系覆盖 | >50% 知识点有 related |
| 列名结构 | Excel第二行列名与模板100%一致 |
