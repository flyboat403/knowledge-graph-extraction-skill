## 认知维度层级

### 布鲁姆教育目标分类法 (Bloom's Taxonomy)

基于修订版布鲁姆分类法的认知层级定义：

| 层级 | 英文名 | 预期行为/产出 | 关键动作 | 学习类型 |
|------|--------|----------------|----------|----------|
| **创造** | Create | 生成创新方案、设计新模式 | 设计、构建、规划、创作 | 高阶思维 |
| **评价** | Evaluate | 进行批判性判断、论证观点 | 判断、选择、评价、辩护 | 高阶思维 |
| **分析** | Analyze | 分解要素、揭示内在关系 | 区分、组织、归因 | 高阶思维 |
| **应用** | Apply | 在新情境中运用知识 | 运用、执行、实施 | 中阶思维 |
| **理解** | Understand | 解释、转换、例证信息 | 解释、示例、分类 | 中阶思维 |
| **记忆** | Remember | 识别、回忆信息 | 识别、回忆 | 低阶思维 |

### 层级递进关系

```
记忆 ← 理解 ← 应用 ← 分析 ← 评价 ← 创造
(下层是上层的基础，上层包含下层能力)
```

## 动词映射规则

### 一、高频映射动词

| 认知层级 | 核心动词（中文） | 对应认知能力 | 雷达词 |
|----------|------------------|--------------|--------|
| **记忆** | 知道、了解、识别、熟悉、记住、说出、回忆 | 信息的再认与回忆 | `了解` `知道` `说出` `回忆` `熟记` |
| **理解** | 理解、解释、说明、描述、归纳、总结、概括 | 信息的解释与转化 | `理解` `解释` `说明` `掌握` |
| **应用** | 掌握、使用、运用、操作、实践、解决 | 知识的迁移与执行 | `使用` `掌握` `应用` `操作` |
| **分析** | 分析、分解、比较、辨析、归因 | 信息的拆解与关系揭示 | `分析` `比较` `分析` `辨析` |
| **评价** | 评价、判断、选择、评估、鉴别 | 价值判断与批判分析 | `评价` `判断` `评估` `鉴别` |
| **创造** | 设计、创造、开发、构建、规划 | 新思路、方案的生成 | `设计` `创造` `构建` `规划` |

### 二、专业术语映射

| 词语类型 | 认知层级 | 说明 |
|----------|----------|------|
| **重点掌握** | 应用 | 强调核心能力，需要实际运用 |
| **必须掌握** | 应用 | 要求达到熟练程度 |
| **熟悉掌握** | 应用 | 不仅要知道还要会使用 |  
| **深刻理解** | 理解 | 程度超过常规理解 |
| **基本理解** | 理解 | 较浅层次但要能解释 |
| **初步了解** | 记忆 | 知道即可 | 
| **大致知道** | 记忆 | 浅层了解 |

## 学科特定层级识别

### 一、理工科领域词汇映射

| 学科领域 | 记忆/理解 | 应用 | 分析 | 评价 | 创造 |
|----------|-----------|------|------|------|------|
| **电工技术** | 了解电路组成 | 能够搭建电路 | 分析电路故障 | 评价电路设计 | 设计最优电路方案 |
| **电子技术** | 识别电子元器件 | 使用元件进行设计 | 解析电路工作原理 | 鉴别方案优劣 | 创新系统架构 | 
| **编程** | 知道语法 | 编写程序 | 排查bug | 评估算法效率 | 创造全新算法 |
| **机械** | 认识零部件 | 会组装调试 | 分析故障原理 | 评判工艺方案 | 创新结构设计 |

### 二、认知层级判断策略

**策略1：任务类型识别**
- "能..." "会..." "操作..." → 应用
- "判断..." "评估..." "评价..." → 评价  
- "设计..." "创建..." "规划..." → 创造
- "分析..." "比较..." "辨别..." → 分析
- "理解..." "掌握..." "说明..." → 理解
- "知道..." "了解..." "说出..." → 记忆

**策略2：工具使用映射**
- 认识工具 → 记忆/理解
- 学会使用 → 应用  
- 排除故障 → 分析/评价
- 创新改进 → 创造

**策略3：情境嵌套识别**
- 静态描述 → 理解/记忆
- 情境变换 → 应用
- 对比分析 → 分析
- 方案优选 → 评价
- 综合创新 → 创造

## 层级判断决策树

```
输入：知识点描述文本

第一步：查找关键词
├ 是否包含创造关键词（"设计"、"创建"、"构建"、"规划"）
│   是 → 【创造】→ 跳转最终确认
└ 否 → 继续第二步

第二步：判断评价意图
├ 是否包含判断/评审含义（"评价"、"判断"、"鉴別"、"选择"）
│   是 → 【评价】→ 跳转最终确认
└ 否 → 继续第三步

第三步：识别分析特征
├ 是否包含拆解/关系揭示（"分析"、"比较"、"对比"、"辨析"）
│   是 → 【分析】→ 跳转最终确认
└ 否 → 继续第四步

第四步：检测应用能力
├ 是否要求使用/操作/实施（"应用"、"使用"、"解决"、"操作"、"掌握" ）
│   是 → 【应用】→ 跳转最终确认
└ 否 → 继续第五步

第五步：确定理解程度
├ 是否表达说明/解释（"理解"、"掌握"、"解释"、"说明"）
│   是 → 【理解】→ 跳转最终确认
└ 否 → 最终确认为【记忆】

第六步：最终确认  
- 检查文本中是否有反向提示（如"重点掌握"虽包含"掌握"但仍归为应用）
- 确认与学科领域匹配度

输出：认知层级
```

| 优先级 | 判断规则 | 示例 |
|--------|----------|------|
| 优先级1 | 包含高级别动词 → 高层级 | "设计并评价方案" → **评价** |

| 优先级4 | 上下文推理判断 | 无明确动词则判断要求深度 |

## 常见层级误判规避

### 混淆点1：动词+对象混淆
❌ **错误判断**："掌握万用表使用" → 理解  
✅ **正解**："使用"表明应操作 → **应用**

### 混淆点2：难度词干扰  
❌ **错误判断**："了解复杂电路设计" → 记忆  
✅ **正解**："设计"明确指向创造 → **创造**

### 混淆点3：修饰词误判  
❌ **错误判断**："深入了解电路原理" → 分析  
✅ **正解**："理解"核心动作，"深入"仅表示程度 → **理解**

### 混淆点4：否定词隐藏意图
❌ **错误判断**："不使用不合理的解决方案" → 记忆/理解
✅ **正解**："不使用+不合理"→ 隐含判断 → **评价**

## 抽取建议策略

### 自动识别规则
1. **高频映射表优先**：直接匹配动词映射表中的明确词条
2. **邻近词分析**：关注动作词与其宾语的语义关系
3. **上下文确认**：结合前缀/后缀词语判断动词真实意图
4. **学科术语校验**：与领域术语对照，确认语义准确性

### 手动校验要点
- [ ] 动词与宾语的语义逻辑是否合理
- [ ] 是否与学科领域的认知习惯匹配  
- [ ] 与前后知识点层级是否连贯
- [ ] 知识点间的认知层级递进关系是否合理

这个体系为自动化抽取提供了标准化的映射规则，既尊重了布鲁姆教育目标分类的经典理论框架，又结合了工程应用实际，使认知层级的自动识别更加精确可靠。
