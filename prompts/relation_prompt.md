# 关系推理 Prompt 模板

## 前置关系推理 Prompt

```
你是知识图谱关系推理专家。请分析以下知识点列表，识别学习依赖关系。

## 知识点列表
{knowledge_nodes}

## 任务

对于每个知识点，判断：
1. 学习这个知识点之前，必须先掌握哪些其他知识点？
2. 这些前置关系是否符合学习逻辑？

## 输出格式

```json
{
  "知识点名称": {
    "pre_requisites": ["前置知识点1", "前置知识点2"],
    "reasoning": "推理依据（简述为什么需要此前置）"
  }
}
```

## 前置关系识别规则

### 1. 概念依赖
- 基础概念是高级概念的前置
- 例：电阻 → 欧姆定律（理解欧姆定律需要先理解电阻）
- 例：电压 → 电路分析（电路分析涉及电压概念）

### 2. 理论-实践依赖
- 理论知识是实践技能的前置
- 例：电路原理 → 电路安装（安装电路需要理解原理）
- 例：编程语法 → 程序开发（开发需要掌握语法）

### 3. 工具-应用依赖
- 工具使用是应用操作的前置
- 例：万用表使用 → 电流测量（测量电流需要会用万用表）
- 例：示波器操作 → 波形分析（分析波形需要会用示波器）

### 4. 逻辑顺序依赖
- 章节间的前后依赖
- 例：基本电路 → 复杂电路分析
- 例：直流电路 → 交流电路

### 5. 组成依赖
- 部分是整体的前置
- 例：电阻、电容、电感 → RLC电路分析

## 注意事项

1. **不要建立冗余关系**：如果 A→B→C，则不需要 A→C（传递性）
2. **区分前置与关联**：
   - 前置：必须先学才能学当前知识点
   - 关联：相关但可以独立学习
3. **考虑实际学习路径**：基于教育实践，而非纯逻辑推导

请开始推理：
```

## 关联关系推理 Prompt

```
你是知识图谱关系推理专家。请分析以下知识点列表，识别知识关联关系。

## 知识点列表
{knowledge_nodes}

## 任务

对于每个知识点，判断：
1. 哪些知识点与它相关但不构成前置/后置关系？
2. 这些关联是否符合知识结构？

## 输出格式

```json
{
  "知识点名称": {
    "related": ["关联知识点1", "关联知识点2"],
    "relation_type": "同类并列/对比概念/相似原理/应用场景关联"
  }
}
```

## 关联关系识别规则

### 1. 同类并列关系
同一主题下的并列概念
- 例：电阻 ↔ 电容 ↔ 电感（都是电路元件）
- 例：电压 ↔ 电流（都是基本电量）

### 2. 对比概念关系
具有对比、对照性质的概念
- 例：直流电路 ↔ 交流电路
- 例：串联 ↔ 并联
- 例：模拟信号 ↔ 数字信号

### 3. 相似原理关系
原理或方法相似的知识点
- 例：欧姆定律 ↔ 基尔霍夫定律（都是电路定律）
- 例：戴维南定理 ↔ 诺顿定理（等效变换方法）

### 4. 应用场景关联
在同一应用场景中共同出现的知识
- 例：变压器 ↔ 电动机（都是电力设备）
- 例：放大电路 ↔ 振荡电路（都是电子电路）

### 5. 扩展延伸关系
知识点间的延伸、扩展关系
- 例：电阻定律 ↔ 欧姆定律（电阻定律是欧姆定律的基础扩展）

## 注意事项

1. **关联关系是无向的**：A 关联 B 等价于 B 关联 A
2. **关联不同于前置**：
   - 关联：可以独立学习，学其中一个不影响学另一个
   - 前置：有严格的学习顺序依赖
3. **避免过度关联**：只关联有实际教学意义的知识点

请开始推理：
```

## 批量关系推理 Prompt

```
你是知识图谱关系推理专家。请为以下知识点同时生成前置和关联关系。

## 知识点列表
{knowledge_nodes}

## 输出格式

```json
[
  {
    "name": "知识点名称",
    "pre_requisites": ["前置知识点1"],
    "related": ["关联知识点1", "关联知识点2"],
    "relation_notes": "关系推理简要说明"
  }
]
```

## 关系生成原则

| 关系类型 | 判断标准 | 示例 |
|----------|----------|------|
| 前置 | 学A之前必须先学B | 电阻 → 欧姆定律 |
| 关联 | 相关但可独立学习 | 电阻 ↔ 电容 |

## 检查清单

- [ ] 前置关系符合学习逻辑
- [ ] 关联关系有实际教学意义
- [ ] 没有冗余关系（如传递性冗余）
- [ ] 每个知识点的关系数量合理（建议≤5个）

请开始推理：
```
