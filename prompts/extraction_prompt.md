# 知识节点抽取 Prompt 模板

## 核心抽取 Prompt

```
你是一个知识图谱构建专家。请从以下文档中抽取知识节点和关系。

## 输入文档
{document_content}

## 输出格式要求

返回 JSON 数组，每个元素代表一个知识节点：

```json
[
  {
    "name": "知识点名称",
    "level": 5,
    "cognitive_level": "理解",
    "category": "概念性",
    "pre_requisites": ["前置知识点1"],
    "related": ["关联知识点1"],
    "tags": "重点;考点",
    "objective": "教学目标描述",
    "description": "知识点简要说明"
  }
]
```

## ⚠️ 关键：节点输出顺序要求

**必须按层级遍历顺序（深度优先）输出节点**：

1. 父节点必须在其所有子节点之前
2. 同一层级的兄弟节点按文档中的出现顺序排列
3. 一个分支的所有节点输出完毕后，再输出下一个分支

**正确示例**：
```
顺序: 课程(A列) → 模块(B列) → 章节(C列) → 章节的子知识点 → 下一个章节 → ...

例：
1. 电子信息类职业技能考试 (level 1)
2.   专业知识(应知) (level 2)
3.     电工技术基础与技能 (level 3)
4.       安全用电常识 (level 4)
5.         触电的种类和形式 (level 5)
6.         触电预防措施 (level 5)
7.       电路基础 (level 4)
8.         电阻 (level 5)
9.         欧姆定律 (level 5)
10.    电子技术基础与技能 (level 3)
11.      ...
12.  技能操作(应会) (level 2)
13.    ...
```

**错误示例（禁止）**：
```
❌ 按层级排序输出（所有level 1 → 所有level 2 → ...）
❌ 子节点出现在父节点之前
❌ 不同分支的节点穿插输出
```

## 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 知识点名称（简洁准确，不超过30字） |
| level | int | 是 | 层级 1-7（1=课程级，7=原子知识点） |
| cognitive_level | string | 是 | 认知层级：记忆/理解/应用/分析/评价/创造 |
| category | string | 是 | 知识分类：事实性/概念性/程序性/元认知 |
| pre_requisites | array | 否 | 前置知识点名称列表（学习依赖，非层级关系） |
| related | array | 否 | 关联知识点名称列表（相关但可独立学习） |
| tags | string | 否 | 标签，多个用英文分号分隔 |
| objective | string | 否 | 教学目标（学习该知识点后应达成的能力或理解） |
| description | string | 否 | 知识点简要说明（不超过100字） |

## 层级分配规则

| level | 对应列 | 典型内容 | 示例 |
|-------|--------|----------|------|
| 1 | A列 | 课程/学科名称 | 电子信息类职业技能考试 |
| 2 | B列 | 模块/单元 | 专业知识(应知) |
| 3 | C列 | 章节/部分 | 电工技术基础与技能 |
| 4 | D列 | 知识主题 | 安全用电常识 |
| 5 | E列 | 具体知识点 | 触电的种类和形式 |
| 6 | F列 | 知识点细分 | 接触触电的类型 |
| 7 | G列 | 原子内容 | 单相触电的具体特征 |

## 前置关系 vs 层级关系（重要区分）

| 概念 | 定义 | JSON 字段 | 示例 |
|------|------|-----------|------|
| **层级关系** | 知识点的归属关系 | 由输出顺序+level确定 | "电阻" 属于 "电路基础" |
| **前置关系** | 学习依赖关系 | pre_requisites | 学 "欧姆定律" 前要先学 "电阻" |

**注意**：
- 层级关系决定节点在树中的位置（通过输出顺序表达）
- 前置关系决定学习路径（通过 pre_requisites 字段表达）
- 两者可能不同：例如 "欧姆定律" 的父节点是 "电路基础"，但前置关系可能是 "电阻"

## 抽取原则

### 0. ⚠️ 排除内容（禁止抽取）

以下内容属于**教学管理信息**，不属于学科知识，**禁止抽取**：

| 排除类型 | 示例 |
|----------|------|
| 课程管理信息 | 课程性质、课程任务、课程地位、教学学时、学分、证书信息 |
| 教学目标描述 | 课程核心素养、思政目标、素质目标 |
| 教学安排信息 | 教学策略、教学方法、教学条件、教学团队要求、教学要求 |
| 评价标准信息 | 评价原则、评价方式、毕业条件|
| 教材编写要求 | 内容选择、呈现形式、体例结构 |
| 课程资源信息 | 文本资源、数字化资源、设施设备 |
| 校企合作信息 | 企业专家参与、实训基地建设 |

**判断测试**：问自己"学生需要学习这个知识吗？"
- 如果答案是"这是教学安排，不是学习内容" → 不抽取
- 如果答案是"学生需要掌握这个技能/概念" → 抽取

**正确示例**：
- "Word文档的段落格式设置方法" ✅ 学科知识
- "Excel中VLOOKUP函数的使用方法" ✅ 学科知识
- "PPT幻灯片母版的设计原则" ✅ 学科知识

**错误示例（禁止抽取）**：
- "本课程计划教学学时为216学时" ❌ 教学安排
- "教学团队规模：专职教师2名" ❌ 教学管理
- "学业质量分为水平一和水平二" ❌ 评价标准
- "采用任务教学法进行教学" ❌ 教学方法
- "课程培养信息素养目标" ❌ 教学目标

### 1. 知识节点识别

**应该抽取**：学科/专业知识内容
- 具体的概念、定义、原理
- 操作方法、步骤、技巧
- 功能特性、参数配置
- 问题解决策略

**不应抽取的**：
- 考试说明、评分标准等行政内容
- "本章主要内容"等导读性文字
- 过于细碎无法独立存在的内容
- **教学管理信息**（见上方排除列表）

### 2. 层级判断

判断层级的三个问题：
- Q1: 这个知识点属于哪个上级模块？（确定父节点）
- Q2: 这个知识点包含哪些下级内容？（确定子节点）
- Q3: 这个知识点与哪些同级概念并列？（确定兄弟节点）

### 3. 前置关系识别

识别学习依赖，以下情况应建立前置关系：

| 模式 | 示例 |
|------|------|
| 基础概念 → 高级应用 | 电阻 → 欧姆定律 |
| 理论原理 → 实践操作 | 电路原理 → 电路安装 |
| 工具使用 → 应用场景 | 万用表使用 → 电流测量 |

### 4. 关联关系识别

识别相关但不构成依赖的知识点：

| 模式 | 示例 |
|------|------|
| 同类并列 | 电阻 ↔ 电容 ↔ 电感 |
| 对比概念 | 直流电路 ↔ 交流电路 |
| 相似原理 | 欧姆定律 ↔ 基尔霍夫定律 |

### 5. 认知层级判断

| 认知层级 | 判断依据 | 典型动词 |
|----------|----------|----------|
| 记忆 | 识别、回忆信息 | 了解、知道、说出 |
| 理解 | 解释、说明含义 | 理解、解释、说明 |
| 应用 | 在新情境中使用 | 掌握、能够、会、操作 |
| 分析 | 分解要素、揭示关系 | 分析、比较、辨析 |
| 评价 | 做出判断、论证观点 | 评价、判断、选择 |
| 创造 | 生成新方案 | 设计、创建、构建 |

## 输出检查清单

抽取完成后，请检查：

- [ ] **节点顺序**：父节点在子节点之前，按深度优先顺序
- [ ] 每个知识点名称简洁准确（不超过30字）
- [ ] 层级正确反映知识的从属关系
- [ ] 前置关系符合学习逻辑
- [ ] 认知层级与内容深度匹配
- [ ] 已过滤非知识内容（考试说明、行政信息）

请开始抽取：

## 简化版 Prompt（快速抽取）

```
从以下文档中抽取知识节点，返回 JSON 数组：

{document_content}

输出格式：
```json
[{"name": "知识点", "level": 1-7, "cognitive_level": "记忆/理解/应用/分析/评价/创造", "category": "事实性/概念性/程序性/元认知", "pre_requisites": [], "related": [], "objective": "教学目标", "description": "知识点说明"}]
```

⚠️ 关键要求：
1. **按深度优先顺序输出**：父节点 → 子节点 → 下一个兄弟节点
2. level 1=课程，2=模块，3=章节，4=主题，5=知识点，6-7=细分
3. pre_requisites 是学习依赖（学B前先学A）
4. related 是相关但可独立学习的知识点

⚠️ 排除内容（禁止抽取）：
- 教学管理信息：课程性质、教学学时、学分、教学团队
- 教学目标：核心素养、知识目标、能力目标
- 评价标准：学业质量、评价原则、评价方式
- 教学安排：教学策略、教学方法、教学条件
- 仅抽取学科知识本身，不抽取"关于课程的信息"

示例输出顺序：
课程(level1) → 模块(level2) → 章节(level3) → 主题(level4) → 知识点(level5) → 下一个主题...
```
