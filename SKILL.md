# 文档知识图谱结构化抽取

## 功能概述

从 docx/pdf 文档中抽取层次化知识节点，标注认知层级和语义关系，输出可直接导入知识图谱系统的结构化文件。

**核心能力：**
- 抽出至少 7 级层次化知识节点
- 标注认知层级（记忆/理解/应用/分析/评价/创造）
- 识另语义关系（属于/依赖于/先修于/关联于等）
- 输出 CSV 和 Excel 格式文件

## 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| source_document | file | 是 | 待抽取的源文档，支持 docx/pdf 格式 |
| template_xlsx | file | 是 | 字段格式模板 xlsx 文件，需读取 A1 单元格的格式要求 |

## 使用步骤

1. 解析模板文件，读取 A1 单元格的格式要求
2. 确定源文档的内容主题，从源文档提取与主题相关的知识内容（须排除教学要求、考试说明等与主题知识无关的内容）并进行结构化分析
3. 抽取层次化知识节点（至少 7 级），标注认知层级
4. 识别节店间语义关系
5. 按模板字段格式生成 CSV 文件
6. 将 CSV 转换为 Excel 文件
7. 输出抽取结果概要

## 输出格式约束（必须严格遵守）

### 列结构定义

| 列范围 | 用途 | 约束 |
|--------|------|------|
| A-G 列 | 知识点层级关系 | 每行只能填写一个知识点，按层级从左到右填充 |
| H-J 列 | 知识点关系 | 填写前置、后置、关联知识点，多个用 `;` 分隔 |
| 后续列 | 标签、认知维度、分类、说明 | 见下方详细约束 |

### 详细约束规则

**1. 层级关系（A-G列）- 树状结构格式（关键约束）**

A列至G列对应知识点的层级关系，采用**树状结构**表示：

- **每行只能填写一个知识点**（仅一个单元格有值，其他为空）
- 知识点所在的列位置表示其层级（A=一级，B=二级，...，G=七级）
- 通过行的顺序和缩进位置形成父子节点关系
- 知识点从上到下按层级展开，形成树状结构

**正确示例：**
```
行1: ['电子信息类职业技能考试', '', '', '', '', '', '']           # 一级知识点
行2: ['', '专业知识(应知)', '', '', '', '', '']      # 二级知识点（电子信息类职业技能考试的子节点）
行3: ['', '', '电工技术基础与技能', '', '', '', '']  # 三级知识点（专业知识(应知)的子节点）
行4: ['', '', '', '安全用电常识', '', '', '']        # 四级知识点（电工技术基础与技能的子节点）
行5: ['', '', '', '', '触电种类和形式', '', '']      # 五级知识点（安全用电常识的子节点）
```

**错误示例（禁止）：**
```
['电子信息类', '专业知识(应知)', '电工技术基础', '', '', '', '']  # ❌ 一行填写多个层级
```

**层级对应关系：**
| 列 | 层级 | 说明 |
|----|------|------|
| A | 一级 | 最高层级（如：课程名称） |
| B | 二级 | A的子节点（如：模块名称） |
| C | 三级 | B的子节点（如：章节名称） |
| D | 四级 | C的子节点（如：知识点主题） |
| E | 五级 | D的子节点（如：具体知识点） |
| F | 六级 | E的子节点 |
| G | 七级 | F的子节点 |

**2. 知识点关系（H-J列）- 必须全部生成**

知识点之间存在三种关系，必须根据文档内容自动识别并生成：

| 列 | 关系类型 | 说明 | 生成规则 |
|----|----------|------|----------|
| H | 前置知识节点 | 学习当前知识点前需要先掌握的知识点 | 基于知识点的逻辑依赖关系识别 |
| I | 后置知识节点 | 学习当前知识点后可以学习的知识点 | H列的反向关系（A→B 则 B的后置是A） |
| J | 关联知识节点 | 与当前知识点相关但不构成前置/后置关系的知识点 | 同类主題、相似概念、对比知识点 |

**关系生成规则：**

1. **前置知识节点（H列）**：
   - 基础概念是高级概念的前置（如：电阻 → 欧姆定律）
   - 工具使用是应用操作的前置（如：万用表使用 → 电流测量）
   - 理论知识是实践技能的前置（如：电路原理 → 电路安装）

2. **后置知识点（I列）**：
   - 自动由前置关系生成：如罘 A 的前置是 B，则 B 的后置是 A
   - 确保关系的双向性

3. **关联知识点（J列）**：
   - 同一主题下的并列知识节点
   - 相似概念或对别概念
   - 不同章节但逻辑相关的知识节点

**格式要求：**
- 多个知识节点之间用英文分号 `;` 隔开
- **重要**：任意两节点之间只能存在一种关系（前置、后置或关联）
- 新导入的关系将覆盖旧关系
- 关系设置总数上限：2000 个

**3. 标签字段**
- 固定标签：`重点`、`难点`、`考点`、`课程思政`
- 可自定义标签
- 多个标签用英文分号 `;` 隔开

**4. 认知维度（必填，单选）**
- 可选值：`记忆`、`理解`、`应用`、`分析`、`评价`、`创造`
- 每个知识节点只能填入一个认知维度

**5. 知识点分类（必填，单选）**
- 可选值：`事实性`、`概念性`、`程序性`、`元认知`
- 每个知识节点只能填入一个分类

**6. 知识点说明**
- 仅支特纯文本输入
- 不支持图片、公式等富媒体内容

**7. 模板完整性**
- 不得删除模板中的任何列
- 不得删除表头行

### 认知层级说明

| 層级 | 说明 |
|------|------|
| 记亿 | 识别、回忆事实性知识 |
| 理解 | 解释、举例、分类 |
| 应用 | 在新情景中使用知识 |
| 分析 | 分解、辨析关系 |
| 评价 | 做出判断、论证观点 |
| 创造 | 生成新观点、设计方案 |

### 语义关系类型

| 关糹类型 | 说明 |
|----------|------|
| 属于 | 知识点的归属关系 |
| 依存于 | 前置知识依赖 |
| 先修于 | 学习顺序关系 |
| 關联于 | 相关性连接 |
| 包含 | 整体与部分关系 |
| 组成 | 构成关系 |
| 因果 | 因果逻辑关系 |

## 关键Anti-Patterns (重要)

**绝对禁止做这些 — 会导致导入失败:**

### 1. NEVER 一行填写多个知识节点

```
❌ 错误: ['一级节点', '二级节点', '三级节点', '', '', '', '']
✅ 正确: ['一级节点', '', '', '', '', '', '']  # 每行只有一个节点
```

**Why**: 导入系统使用树结构。每行表示 ONE 节点。位置表示层级。

### 2. NEVER 使用中文分号分隔关系

```
❌ 错误: 知识节点A；知识节点B；知识节点C
✅ 正确: 知识节点A;知识节点B;知识节点C
```

**Why**: 解析器期待 ASCII 分号 (`;`, U+003B)，不是中文分号 (`；`, U+FF1B)。

### 3. NEVER 填写多个认知维度或分类

```
❌ 错误: L列 = "记忆;理解"
✅ 正确: L列 = "理解"  # 单选值

❌ 错误: M列 = "概念性;程序性"  
✅ 正确: M列 = "概念性"  # 単选值
```

**Why**: 这些字段是单选。多个值会导致导入失败。

### 4. NEVER 删除或重排列模板的列

```
❌ 错误: 删除K列，或调整A-G列顺序
✅ 正确: 保持15列完整 (A-O)
```

**Why**: 导入系统期待固定列位置。缺失列会破坏映射。

### 5. NEVER 不完整的层级路径

```
❌ 错误: 
  行1: [A列: 课程] 
  行2: [C列: 章节]  # 跳过了B列

✅ 正确:
  行1: [A列: 课程]
  行2: [B列: 模块]
  行3: [C列: 章节]
```

**Why**: 每个知识节点必须有完整的父链。跳躍层级会破坏树结构。

## 抽取决策框架

抽取前问自己以下问题:

### 1. 边界判断

**问题**: 这个节点能独立存在吗？

| 情况 | 判断 | 行动 |
|------|------|------|
| 概念过大 ("电路基础") | 太宽 | 拆分为多个子节点 |
| 细节过碎 ("电阻的单位") | 太窄 | 合并到上级节点 |
| 可独立理解 | 合適 | 保留为独立节点 |

**判断标准**: 如果去掉上下文，这个节点是否有独立意义？

### 2. 层级定位

**问自己三个问题:**

```
Q1: 上级 - 这个节点属于哪个模块/章节？
Q2: 下级 - 这个节点包含哪些具体内容？  
Q3: 平级 - 还有哪些同类节点？
```

**层级分配规则:**
| 内容类型 | 典型层级 |
|----------|----------|
| 课程/学科名称 | A列 (一级) |
| 模块/单元 | B列 (二级) |
| 章节/部分 | C列 (三级) |
| 知识主题 | D列 (四级) |
| 具体知识点 | E列 (五级) |

### 3. 关系识别

**前置关系 (H列) - 必须先掌握的:**
- 基础概念 → 高级概念 (电阻 → 欧姆定律)
- 工具使用 → 应用操作 (万用表 → 测量电流)
- 理论知识 → 实践技能 (电路原理 → 电路安装)

**后置关系 (I列) - 学完后可学的:**
- 自动生成: 如果 A的前置是B，则 B的后置是A
- 确保双向性

**关联关系 (J列) - 相关但非依赖:**
- 同一主题下的并列节点
- 相似或对比概念
- 不同章节但逻辑相关的节点

## 工作流程

### 步骤 1: 读取模板规则

**MANDATORY - 阅读整个文件**: 在处理前，必须完整阅读 [references/output-format.md](references/output-format.md) 以了解确切的列結構约束。

使用脚本自动解析模板:
```bash
python scripts/extract_knowledge_graph.py --template template.xlsx --dry-run
```

### 步骤 2: 解析源文档

使用脚本处理，脚本已内置格式选择逻辑:
```bash
# docx 文档
python scripts/extract_knowledge_graph.py --source document.docx --template template.xlsx --output output.xlsx

# pdf 文档  
python scripts/extract_knowledge_graph.py --source document.pdf --template template.xlsx --output output.csv
```

### 步骤 3: 构建知识节点树

**MANDATORY - 读取完整文件**: 在构建前，阅读 [references/output-format.md](references/output-format.md) 中的 "树状结构约束" 部分。

核心规则: **每行一个节点**，位置表示层级。

### 步骤 4: 生成关系

根据"抽取决策框架"中的关系识别规则，为每个节点填充 H/I/J 列。

### 步骤 5: 验证與输出

脚本自动验证:
- 每行只有一个节点 (A-G列)
- 认知维度和分类为单选值
- 分隔符为英文分号

## 输出产物

1. **extracted_csv** - 结构化抽取结果的 CSV 文件（逗号分隔，可直接导入知识图谱工具）
2. **converted_xlsx** - 由 CSV 转换后的 Excel 文件
3. **extraction_summary** - 抽取结果概要（节点数量、层级分布、关系类型等）

## 兼容性

- 可导入平台：Neo4j、Apache Jena、百度百科图谱
- 源文档格式：docx、pdf
- 模版格式：xlsx

## 错误处理

| 错误代码 | 说明 |
|----------|------|
| FILE_FORMAT_ERROR | 上传的文档格式不支持，仅支持 docx/pdf（源文档）和 xlsx（模版） |
| CELL_READ_ERROR | 无法读取 xlsx 模版 A1 单元格，请检查文件是否损坏 |
| HIERARCHY_INSUFFICIENT | 文档中提取的知识节点层级不足 6 級，请补充文档内容 |
