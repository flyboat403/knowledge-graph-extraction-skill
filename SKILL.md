# 文档知识图谱结构化抽取

## 功能概述

从 docx/pdf 文档中抽取层次化知识节点，标注认知层级和语义关系，输出可直接导入知识图谱系统的结构化文件。

**核心能力：**
- 抽出至少 7 级层次化知识节点
- 标注认知层级（记忆/理解/应用/分析/评价/创造）
- 识别语义关系（前置/后置/关联）
- 输出 CSV 和 Excel 格式文件

**技术架构：**

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   文档解析    │───→│  LLM 语义抽取 │───→│   格式验证    │───→│  Excel 生成  │
│   (脚本)     │    │   (LLM)      │    │   (脚本)     │    │   (脚本)     │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
      │                    │                   │                   │
      ▼                    ▼                   ▼                   ▼
  提取纯文本          语义理解+推理        验证格式约束         生成标准输出
  解析模板           生成节点+关系        修复格式错误
```

## 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| source_document | file | 是 | 待抽取的源文档，支持 docx/pdf 格式 |
| template_xlsx | file | 是 | 字段格式模板 xlsx 文件，需读取 A1 单元格的格式要求 |

## 使用步骤

### 步骤 1: 解析模板规则

**MANDATORY - 阅读整个文件**: 使用脚本解析模板，获取格式约束：

```bash
python scripts/extract_knowledge_graph.py --template template.xlsx --dry-run
```

**MANDATORY - 读取完整文件**: 阅读 [references/output-format.md](references/output-format.md) 了解列结构约束。

### 步骤 2: LLM 语义化知识抽取（核心步骤）

**此步骤由 LLM 执行语义理解和知识抽取。**

#### 2.1 读取 Prompt 模板

**MANDATORY - 读取完整文件**: 阅读 [prompts/extraction_prompt.md](prompts/extraction_prompt.md) 获取完整 Prompt 模板。

#### 2.2 执行 LLM 抽取

使用以下 Prompt 模板进行知识抽取：

```
你是一个知识图谱构建专家。请从以下文档中抽取知识节点和关系。

## 输入文档
{document_content}

## 输出格式要求

返回 JSON 数组，每个元素代表一个知识节点：

```json
[
  {
    "name": "知识点名称",
    "level": 5,
    "cognitive_level": "理解",
    "category": "概念性",
    "pre_requisites": ["前置知识点1"],
    "related": ["关联知识点1"],
    "tags": "重点;考点",
    "description": "知识点简要说明"
  }
]
```

## ⚠️ 关键：节点输出顺序要求

**必须按层级遍历顺序（深度优先）输出节点**：

1. 父节点必须在其所有子节点之前
2. 同一层级的兄弟节点按文档中的出现顺序排列
3. 一个分支的所有节点输出完毕后，再输出下一个分支

**正确示例**：
```
1. 课程 (level 1)
2.   模块A (level 2)
3.     章节A1 (level 3)
4.       主题A1a (level 4)
5.         知识点A1a1 (level 5)
6.         知识点A1a2 (level 5)
7.       主题A1b (level 4)
8.     章节A2 (level 3)
9.   模块B (level 2)
10.    ...
```

**错误示例（禁止）**：按层级排序（所有level 1 → 所有level 2 → ...）

## 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 知识点名称（简洁准确，不超过30字） |
| level | int | 是 | 层级 1-7（1=课程，2=模块，3=章节，4=主题，5=知识点，6-7=细分） |
| cognitive_level | string | 是 | 认知层级：记忆/理解/应用/分析/评价/创造 |
| category | string | 是 | 知识分类：事实性/概念性/程序性/元认知 |
| pre_requisites | array | 否 | 前置知识点名称列表（学习依赖，非层级关系） |
| related | array | 否 | 关联知识点名称列表（相关但可独立学习） |
| tags | string | 否 | 标签，多个用英文分号分隔 |
| description | string | 否 | 知识点简要说明 |

## 前置关系 vs 层级关系

| 概念 | 定义 | 示例 |
|------|------|------|
| **层级关系** | 知识点的归属关系 | "电阻" 属于 "电路基础" |
| **前置关系** | 学习依赖关系 | 学 "欧姆定律" 前要先学 "电阻" |

## 抽取原则

### 0. ⚠️ 排除内容（禁止抽取）

以下内容属于**教学管理信息**，不属于学科知识，**禁止抽取**：

| 排除类型 | 示例 |
|----------|------|
| 课程管理信息 | 课程性质、课程任务、教学学时、学分 |
| 教学目标描述 | 课程核心素养、素质目标 |
| 教学安排信息 | 教学策略、教学方法、教学条件、教学要求 |
| 评价标准信息 | 评价原则、评价方式 |

**判断测试**：问自己"学生需要学习这个知识吗？"
- "这是教学安排，不是学习内容" → 不抽取
- "学生需要掌握这个技能/概念" → 抽取

**正确示例**：Word排版方法 ✅、Excel函数使用 ✅、PPT设计原则 ✅

**错误示例**：教学学时216学时 ❌、教学团队2名教师 ❌、学业质量评价 ❌

### 1. 抽取焦点

**应该抽取**：学科/专业知识内容
- 具体的概念、定义、原理
- 操作方法、步骤、技巧
- 功能特性、参数配置

2. **节点顺序**：按深度优先顺序输出（父→子→下一个兄弟）
3. **层级判断**：level 反映知识的从属关系
4. **前置关系**：识别学习依赖（基础→高级，理论→实践）
5. **关联关系**：识别相关概念（同类并列、对比概念）
6. **过滤非知识内容**：排除教学管理信息、考试说明等

请开始抽取：
```

#### 2.3 LLM 抽取检查清单

抽取完成后，LLM 应自检：

- [ ] **节点顺序**：父节点在子节点之前，按深度优先顺序
- [ ] **排除教学管理信息**：未抽取课程性质、教学学时、评价标准等
- [ ] 每个知识点名称简洁准确（不超过30字）
- [ ] 层级正确反映知识的从属关系
- [ ] 前置关系符合学习逻辑
- [ ] 认知层级与内容深度匹配
- [ ] 已过滤非知识内容

### 步骤 3: 关系推理（可选增强）

如需更精准的关系推理，使用 [prompts/relation_prompt.md](prompts/relation_prompt.md) 进行关系推理。

**前置关系识别规则：**

| 模式 | 示例 |
|------|------|
| 基础概念 → 高级应用 | 电阻 → 欧姆定律 |
| 理论原理 → 实践操作 | 电路原理 → 电路安装 |
| 工具使用 → 应用场景 | 万用表使用 → 电流测量 |

**关联关系识别规则：**

| 模式 | 示例 |
|------|------|
| 同类并列 | 电阻 ↔ 电容 ↔ 电感 |
| 对比概念 | 直流电路 ↔ 交流电路 |
| 相似原理 | 欧姆定律 ↔ 基尔霍夫定律 |

### 步骤 4: 格式验证与 Excel 生成

将 LLM 输出的 JSON 数据传入脚本，生成 Excel 文件：

```bash

  --json knowledge_nodes.json \
  --template template.xlsx \
  --output output.xlsx
```

脚本自动验证：
- 每行只有一个节点 (A-G列)
- 认知维度和分类为单选值
- 分隔符为英文分号
- 树状结构完整性

### 步骤 5: 输出结果

1. **output.xlsx** - 结构化抽取结果的 Excel 文件
2. **output.csv** - CSV 格式备份（可直接导入知识图谱工具）
3. **抽取统计** - 节点数量、层级分布、关系类型等

## 输出格式约束（必须严格遵守）

### 列结构定义

| 列范围 | 用途 | 约束 |
|--------|------|------|
| A-G 列 | 知识点层级关系 | 每行只能填写一个知识点，按层级从左到右填充 |
| H-J 列 | 知识点关系 | 填写前置、后置、关联知识点，多个用 `;` 分隔 |
| 后续列 | 标签、认知维度、分类、说明 | 见下方详细约束 |

### 详细约束规则

**1. 层级关系（A-G列）- 树状结构格式（关键约束）**

- **每行只能填写一个知识点**（仅一个单元格有值，其他为空）
- 知识点所在的列位置表示其层级（A=一级，B=二级，...，G=七级）
- 通过行的顺序和缩进位置形成父子节点关系

**正确示例：**
```
行1: ['电子信息类职业技能考试', '', '', '', '', '', '']           # 一级
行2: ['', '专业知识(应知)', '', '', '', '', '']                     # 二级
行3: ['', '', '电工技术基础与技能', '', '', '', '']                  # 三级
行4: ['', '', '', '安全用电常识', '', '', '']                        # 四级
行5: ['', '', '', '', '触电种类和形式', '', '']                      # 五级
```

**错误示例（禁止）：**
```
['电子信息类', '专业知识(应知)', '电工技术基础', '', '', '', '']  # ❌ 一行填写多个层级
```

**2. 知识点关系（H-J列）- 必须全部生成**

| 列 | 关系类型 | 说明 | 生成规则 |
|----|----------|------|----------|
| H | 前置知识节点 | 学习当前知识点前需要先掌握的知识点 | LLM 语义推理 |
| I | 后置知识节点 | 学习当前知识点后可以学习的知识点 | H列的反向关系 |
| J | 关联知识节点 | 与当前知识点相关但不构成前置/后置关系 | LLM 语义推理 |

**格式要求：**
- 多个知识节点之间用英文分号 `;` 隔开
- **重要**：任意两节点之间只能存在一种关系（前置、后置或关联）
- 关系设置总数上限：2000 个

**3. 认知维度（必填，单选）**
- 可选值：`记忆`、`理解`、`应用`、`分析`、`评价`、`创造`
- 每个知识节点只能填入一个认知维度

**4. 知识点分类（必填，单选）**
- 可选值：`事实性`、`概念性`、`程序性`、`元认知`
- 每个知识节点只能填入一个分类

## 关键 Anti-Patterns（重要）

**绝对禁止做这些 — 会导致导入失败：**

### 1. NEVER 一行填写多个知识节点

```
❌ 错误: ['一级节点', '二级节点', '三级节点', '', '', '', '']
✅ 正确: ['一级节点', '', '', '', '', '', '']  # 每行只有一个节点
```

**Why**: 导入系统使用树结构。每行表示 ONE 节点。位置表示层级。

### 2. NEVER 使用中文分号分隔关系

```
❌ 错误: 知识节点A；知识节点B；知识节点C
✅ 正确: 知识节点A;知识节点B;知识节点C
```

**Why**: 解析器期待 ASCII 分号 (`;`, U+003B)，不是中文分号 (`；`, U+FF1B)。

### 3. NEVER 填写多个认知维度或分类

```
❌ 错误: L列 = "记忆;理解"
✅ 正确: L列 = "理解"  # 单选值
```

**Why**: 这些字段是单选。多个值会导致导入失败。

### 4. NEVER 不完整的层级路径

```
❌ 错误: 
  行1: [A列: 课程] 
  行2: [C列: 章节]  # 跳过了B列

✅ 正确:
  行1: [A列: 课程]
  行2: [B列: 模块]
  行3: [C列: 章节]
```

**Why**: 每个知识节点必须有完整的父链。跳跃层级会破坏树结构。

### 5. NEVER 遗漏关联关系

```
❌ 错误: J列全部为空
✅ 正确: J列包含语义相关的知识点（同类、对比、相似原理）
```

**Why**: 关联关系是知识图谱的重要组成部分，缺失会降低图谱质量。

## 抽取决策框架

### LLM 抽取时应遵循的决策逻辑：

#### 1. 边界判断

**问题**: 这个节点能独立存在吗？

| 情况 | 判断 | 行动 |
|------|------|------|
| 概念过大 ("电路基础") | 太宽 | 拆分为多个子节点 |
| 细节过碎 ("电阻的单位") | 太窄 | 合并到上级节点 |
| 可独立理解 | 合适 | 保留为独立节点 |

#### 2. 层级定位

**问自己三个问题：**

```
Q1: 上级 - 这个节点属于哪个模块/章节？
Q2: 下级 - 这个节点包含哪些具体内容？  
Q3: 平级 - 还有哪些同类节点？
```

#### 3. 关系识别

**前置关系 (H列)**：
- 基础概念 → 高级概念 (电阻 → 欧姆定律)
- 工具使用 → 应用操作 (万用表 → 测量电流)
- 理论知识 → 实践技能 (电路原理 → 电路安装)

**关联关系 (J列)**：
- 同一主题下的并列节点 (电阻 ↔ 电容 ↔ 电感)
- 相似或对比概念 (直流电路 ↔ 交流电路)
- 不同章节但逻辑相关的节点

#### 4. 认知层级判断

| 认知层级 | 判断依据 | 典型动词 |
|----------|----------|----------|
| 记忆 | 识别、回忆信息 | 了解、知道、说出 |
| 理解 | 解释、说明含义 | 理解、解释、说明 |
| 应用 | 在新情境中使用 | 掌握、能够、会、操作 |
| 分析 | 分解要素、揭示关系 | 分析、比较、辨析 |
| 评价 | 做出判断、论证观点 | 评价、判断、选择 |
| 创造 | 生成新方案 | 设计、创建、构建 |

## 错误处理

| 错误代码 | 说明 |
|----------|------|
| FILE_FORMAT_ERROR | 上传的文档格式不支持 |
| JSON_PARSE_ERROR | LLM 输出的 JSON 格式无效 |
| HIERARCHY_INSUFFICIENT | 知识节点层级不足 6 级 |
| RELATION_MISSING | 关联关系(J列)全部为空 |
