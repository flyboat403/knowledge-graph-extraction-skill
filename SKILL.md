# 文档知识图谱结构化抽取

## 功能概述

从 docx/pdf 文档中抽取层次化知识节点，标注认知层级和语义关系，输出可直接导入知识图谱系统的结构化文件。

**核心能力：**
- 抽取至少 6 级层次化知识节点
- 标注认知层级（记忆/理解/应用/分析/评价/创造）
- 识别语义关系（属于/依赖于/先修于/关联于等）
- 输出 CSV 和 Excel 格式文件

## 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| source_document | file | 是 | 待抽取的源文档，支持 docx/pdf 格式 |
| template_xlsx | file | 是 | 字段格式模板 xlsx 文件，需读取 A1 单元格的格式要求 |

## 使用步骤

1. 解析模板文件，读取 A1 单元格的格式要求
2. 提取源文档中的文本内容并进行结构化分析
3. 抽取层次化知识节点（至少 6 级），标注认知层级
4. 识别节点间语义关系
5. 按模板字段格式生成 CSV 文件
6. 将 CSV 转换为 Excel 文件
7. 输出抽取结果概要

## 输出格式约束（必须严格遵守）

### 列结构定义

| 列范围 | 用途 | 约束 |
|--------|------|------|
| A-G 列 | 知识点层级关系 | 每行只能填写一个知识点，按层级从左到右填充 |
| H-J 列 | 知识点关系 | 填写前置、后置、关联知识点，多个用 `;` 分隔 |
| 后续列 | 标签、认知维度、分类、说明 | 见下方详细约束 |

### 详细约束规则

**1. 层级关系（A-G列）- 树状结构格式（关键约束）**

A列至G列对应知识点的层级关系，采用**树状结构**表示：

- **每行只能填写一个知识点**（仅一个单元格有值，其他为空）
- 知识点所在的列位置表示其层级（A=一级，B=二级，...，G=七级）
- 通过行的顺序和缩进位置形成父子节点关系
- 知识点从上到下按层级展开，形成树状结构

**正确示例：**
```
行1: ['演示文稿', '', '', '', '', '', '']           # 一级知识点
行2: ['', '新建演示文稿', '', '', '', '', '']        # 二级知识点（演示文稿的子节点）
行3: ['', '', '幻灯片大小', '', '', '', '']          # 三级知识点（新建演示文稿的子节点）
行4: ['', '演示文稿设计', '', '', '', '', '']        # 二级知识点（演示文稿的另一子节点）
行5: ['', '', '主题', '', '', '', '']                # 三级知识点（演示文稿设计的子节点）
```

**错误示例（禁止）：**
```
['演示文稿', '新建演示文稿', '幻灯片大小', '', '', '', '']  # ❌ 一行填写多个层级
```

**层级对应关系：**
| 列 | 层级 | 说明 |
|----|------|------|
| A | 一级 | 最高层级（如：课程名称） |
| B | 二级 | A的子节点（如：模块名称） |
| C | 三级 | B的子节点（如：章节名称） |
| D | 四级 | C的子节点（如：知识点主题） |
| E | 五级 | D的子节点（如：具体知识点） |
| F | 六级 | E的子节点 |
| G | 七级 | F的子节点 |

**2. 知识点关系（H-J列）- 必须全部生成**

知识点之间存在三种关系，必须根据文档内容自动识别并生成：

| 列 | 关系类型 | 说明 | 生成规则 |
|----|----------|------|----------|
| H | 前置知识点 | 学习当前知识点前需要先掌握的知识点 | 基于知识点的逻辑依赖关系识别 |
| I | 后置知识点 | 学习当前知识点后可以继续学习的知识点 | H列的反向关系（A→B 则 B的后置是A） |
| J | 关联知识点 | 与当前知识点相关但不构成前置/后置关系的知识点 | 同类主题、相似概念、对比知识点 |

**关系生成规则：**

1. **前置知识点（H列）**：
   - 基础概念是高级概念的前置（如：电阻 → 欧姆定律）
   - 工具使用是应用操作的前置（如：万用表使用 → 电流测量）
   - 理论知识是实践技能的前置（如：电路原理 → 电路安装）

2. **后置知识点（I列）**：
   - 自动由前置关系生成：如果 A 的前置是 B，则 B 的后置是 A
   - 确保关系的双向性

3. **关联知识点（J列）**：
   - 同一主题下的并列知识点
   - 相似概念或对比概念
   - 不同章节但逻辑相关的知识点

**格式要求：**
- 多个知识点之间用英文分号 `;` 隔开
- **重要**：任意两个知识点之间只能存在一种关系（前置、后置或关联）
- 新导入的关系将覆盖旧关系
- 关系设置总数上限：2000 个

**3. 标签字段**
- 固定标签：`重点`、`难点`、`考点`、`课程思政`
- 可自定义标签
- 多个标签用英文分号 `;` 隔开

**4. 认知维度（必填，单选）**
- 可选值：`记忆`、`理解`、`应用`、`分析`、`评价`、`创造`
- 每个知识点只能填入一个认知维度

**5. 知识点分类（必填，单选）**
- 可选值：`事实性`、`概念性`、`程序性`、`元认知`
- 每个知识点只能填入一个分类

**6. 知识点说明**
- 仅支持纯文本输入
- 不支持图片、公式等富媒体内容

**7. 模板完整性**
- 不得删除模板中的任何列
- 不得删除表头行

### 认知层级说明

| 层级 | 说明 |
|------|------|
| 记忆 | 识别、回忆事实性知识 |
| 理解 | 解释、举例、分类 |
| 应用 | 在新情境中使用知识 |
| 分析 | 分解、辨析关系 |
| 评价 | 做出判断、论证观点 |
| 创造 | 生成新观点、设计方案 |

### 语义关系类型

| 关系类型 | 说明 |
|----------|------|
| 属于 | 知识点的归属关系 |
| 依赖于 | 前置知识依赖 |
| 先修于 | 学习顺序关系 |
| 关联于 | 相关性连接 |
| 包含 | 整体与部分关系 |
| 组成 | 构成关系 |
| 因果 | 因果逻辑关系 |

## 输出产物

1. **extracted_csv** - 结构化抽取结果的 CSV 文件（逗号分隔，可直接导入知识图谱工具）
2. **converted_xlsx** - 由 CSV 转换后的 Excel 文件
3. **extraction_summary** - 抽取结果概要（节点数量、层级分布、关系类型等）

## 兼容性

- 可导入平台：Neo4j、Apache Jena、百度百科图谱
- 源文档格式：docx、pdf
- 模板格式：xlsx

## 错误处理

| 错误代码 | 说明 |
|----------|------|
| FILE_FORMAT_ERROR | 上传的文档格式不支持，仅支持 docx/pdf（源文档）和 xlsx（模板） |
| CELL_READ_ERROR | 无法读取 xlsx 模板 A1 单元格，请检查文件是否损坏 |
| HIERARCHY_INSUFFICIENT | 文档中提取的知识节点层级不足 6 级，请补充文档内容 |
