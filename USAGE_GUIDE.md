# 文档知识图谱抽取技能使用指南

## 🎯 功能概述

本技能可以从 DOCX/PDF 文档中自动抽取层次化的知识节点，标注认知层级和语义关系，生成可用于知识图谱导入的 CSV/Excel 格式文件。

## 📋 用法说明

### 输入要求

- **源文档**：`docx` 或 `pdf` 格式
- **模板文件**：`xlsx` 格式，其中 A1 单元格包含输出格式约束

### 输出格式

生成以下结构的文件：
- **A-G列**：知识节点层级关系（每行只能填写一个节点）
- **H-I-J列**：前置/后置/关联知识点
- **K-O列**：标签、认知等级、分类等元数据

## 💡 使用场景

- 从课程标准抽取知识点构建知识图谱
- 教育领域的结构化教学数据分析
- 自动知识图谱构建与导入
- 教学内容的层次化梳理

## 🔧 使用示例

### 1. 直接运行脚本

```bash
python scripts/extract_knowledge_graph.py \
  --source input_document.docx \
  --template assets/template.xlsx \
  --output output.xlsx
```

### 2. 知识点抽取决策框架

抽取前问自己：

1. **边界判断**：这个知识点能独立存在吗？
   - 太宽泛 → 拆分为多个知识点
   - 太细碎 → 合并到上级节点
   - 适中 → 保留为独立节点

2. **层级定位**：判断知识点应该放置在哪一级别
   - 课程级 → A列
   - 模块级 → B列
   - 章节级 → C列
   - 主题级 → D列
   - 具体知识 → E列

3. **关系识别**：
   - **前置关系**：学这个之前需要掌握的知识（H列）
   - **后置关系**：学完这个之后可学的知识（I列）  
   - **关联关系**：相关但非依赖的知识点（J列）

## ⚠️ 重要约束

### 树状结构约定

- ✅ **正确示例**：
```
['知识点A', '', '', '', '', '', '']        # A级知识点
['', '知识点B', '', '', '', '', '']      # B级知识点(知识点A的子节点)
['', '', '知识点C', '', '', '', '']      # C级知识点(知识点B的子节点)
```

- ❌ **错误示例**：
```
['知识点A', '知识点B', '知识点C', '', '', '', '']  # 多个知识点在同一行
```

### 认知维度定义

| 维度 | 识别词汇 | 说明 |
|------|----------|------|
| 记忆 | 了解、知道、说出 | 识别、回忆事实性知识 |
| 理解 | 理解、明白、解释 | 解释、举例、分类 |
| 应用 | 掌握、会、能够 | 在新情境中使用知识 |
| 分析 | 分析、比较、归纳 | 分解、辨析关系 |
| 评价 | 评价、判断、选择 | 做出判断、论证观点 |
| 创造 | 设计、创造、规划 | 生成新观点、设计方案 |

### 不可违反反模式

1. **绝对不**在一行填写多个知识点
2. **绝对不**使用中文分号`；`，而要使用英文分号`;`
3. **绝对不**在认知维度(L栏)填入多个值
4. **绝对不**删除模板的A-O列中任何一列

## 📈 输出验证

1. **格式验证**：检查每行是否只有一个知识点（A-G列中只有一个非空）
2. **认知验证**：检查L列值是否符合认知维度（记忆/理解/应用/分析/评价/创造）
3. **分类验证**：检查M列值是否符合知识分类（事实性/概念性/程序性/元认知）
