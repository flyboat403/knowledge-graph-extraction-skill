# OpenCode 文档知识图谱抽取技能

> 从 DOCX/PDF 抽取 7+ 级结构化知识点，生成可导入图谱系统的关系数据

---

## 🧩 技能识别参数

| 参数 | 值 |
|------|-----|
| **名称** | knowledge-graph-extractor |
| **触发词** | "文档知识图谱" / "知识点抽取" / "结构化抽取" / "知识图谱构建" |
| **输入格式** | `docx/pdf` 源文档 + `xlsx` 模板文件 |
| **输出格式** | `xlsx/csv` 图谱导入数据 (符合A-O列树状约束) |

---

## ⚙️ 完整使用流程

```
源文档(Word/PDF)  
         ↓
    [技能引擎处理]
         ↓          
    层级结构(A-G列)  
    关系提取(H-I-J列) 
    认知标注(L-M列)
         ↓
输出可导入 Neo4j/Jena 文件 ✅
```

**命令行使用**:
```bash
# 进入安装目录
cd ~/.agents/skills/knowledge-graph-extractor

# 运行抽取
python scripts/extract_knowledge_graph.py \
  --source 教学文档.docx \
  --template assets/template.xlsx \
  --output 知识图谱.xlsx
```

## 📊 A-O列输出格式 (必遵守)

| 列 | 约束 | 说明 | 示例 |
|----|------|------|------|  
| **A-G** | 每行仅1个非空 | 树状层级结构 | 课程→模块→章节→主题→知识点→... |
| **H** | 前置依赖 | 学前须掌握 | 电阻概念;电流概念 |
| **I** | 后续学习 | 学后可学习 | 欧姆定律;基尔霍夫定律 |
| **J** | 关联知识 | 相关知识点 | 电压→电流(关联);电阻 |  
| **K** | 标签 | 重点;难点;考点 | 重点;考点 |
| **L** | 认知维度 | 单选(记忆/理解/应用/...) | 应用/分析/评价 |
| **M** | 知识分类 | 单选(事实/概念/程序/...) | 概念性/程序性 |
| **N-O** | 目标/说明 | 文本描述 | 教学目标/知识点解析 |

💡 **树状结构要求**: 每行只能填1个知识点（A-G列），确保层次清晰

---

## 🧠 认知层级自动识别规则

| 动词类别 | 认知层级 | 典型示例 |
|----------|----------|----------|  
| **掌握/能够/会** | 应用 ✅ | "能够使用万用表测量" |
| **理解/解释/说明** | 理解 | "理解欧姆定律含义" |
| **知道/了解/说出** | 记忆 | "知道电阻单位" |
| **分析/比较/辨析** | 分析 | "分析电路故障" |

---

## 🔗 语义关系智能生成

1. **前置关系 (H列)**: 依赖型关系 (学A前必须先学B)  
   - `A的前置是B` ↔ `B的后置是A`

2. **后置关系 (I列)**: 前置关系的反向自动生成

3. **关联关系 (J列)**: 相关但非依赖 (并列/对照/相似)  

---

## 🎓 典型应用场景

- **课程标准** → 知识图谱 (构建学科知识点地图)  
- **考试大纲** → 重点分析 (提取考点分布/层级关系)
- **教材** → 内容组织 (知识点关联/学习路径设计)
- **培训资料** → 技能评估 (技能点依赖/学习进阶)
