# 发布日志 (Release Notes)

## v1.0.0 - 2026年3月15日

### 🎯 主要功能

首次发布的完整版本，包含文档知识图谱结构化抽取的所有核心功能：

- **层次化抽取**：支持从 docx/pdf 文档提取 7 级以上知识点层级
- **认知标注**：自动识别认知层级（记忆→理解→应用→分析→评价→创造） 
- **关系生成**：自动生成前置/后置/关联关系
- **结构化输出**：生成树状结构的 Excel/CSV 格式文件
- **反模式防护**：内置详尽的防错指南和验证约束

### 🚀 核心能力

| 功能 | 说明 | 规格 |
|------|------|------|
| **文档支持** | 源文档格式 | `docx`, `pdf` |
| **输出格式** | 结构化数据格式 | `xlsx`, `csv` |
| **层级深度** | 知识点层数 | ≥7级 (A-G列) |
| **关系类型** | 前置/后置/关联 | 支持多项分隔 |
| **认知维度** | 布鲁姆分类 | 记忆/理解/应用/分析/评价/创造 |
| **知识分类** | Tetty Lee分类 | 事实性/概念性/程序性/元认知 |
| **兼容平台** | 知识图谱导入 | Neo4j, Jena, 百度图谱 |

### 🛠️ 技术实现

#### 树状结构约束
- 每行只填写一个知识点 (A-G 列区间仅一个非空) ✅
- 前置/后置/关联关系自动生成 (H-I-J 列) ✅  
- 雷达词分隔符英文分号 (`;`) ✅
- 固定模板列保留 (A-O 列完整) ✅

#### 代码架构
```
SKILL.md          # 核心技能定义和触发条件
scripts/
├── extract_knowledge_graph.py   # 文档解析+知识抽取+关系生成
references/
├── output-format.md             # 列格式约束定义
├── cognitive-levels.md          # 认知维度判断指南
├── knowledge-categories.md      # 知识类型识别规则
assets/
└── template.xlsx                # 输出模板文件
examples/
├── input/                       # 示例输入文档
└── output/                      # 示例输出结果
```

#### 关系生成策略
1. **前置关系**(H列)：逻辑依赖（电阻 → 欧姆定律）
2. **后置关系**(I列)：前置逆向（欧姆定律 → 电阻）  
3. **关联关系**(J列)：同类/対比/并列（电压 ↔ 电流）

### 🧠 决策框架

#### 边界判断
- 概念过大：拆分为多个子节点
- 细节过碎：合并到上级节点  
- 独立存在：保留为节点

#### 层级定位  
- 一级：学科/课程名称
- 二级：模块/单元
- 三级：章节/部分
- 四级：主题/知识点组
- 五级：具体知识点
- 六~七级：扩展/细化知识点

#### 关系识别
- 基础 → 高级：前置关系
- 工具使用 → 操作应用：前置关系
- 並列关系：关联关系

### 💻 安装与使用

#### 1. 通过 OpenCode CLI 安装：
```bash
opencode skills add https://github.com/flyboat403/knowledge-graph-extraction-skill.git
```

#### 2. 命令行使用：
```bash
python scripts/extract_knowledge_graph.py \
  --source 课程标准.docx \
  --template assets/template.xlsx \
  --output output_result.xlsx
```

#### 3. 输入示例：
- 源文档：05电子信息类.docx
- 模板：sample_template.xlsx
