# 文档知识图谱抽取技能说明书

## 作用
从教育文档（如课程标准、考试大纲）中抽取层次化知识节点，自动生成知识点关系，输出适用于知识图谱导入的结构化数据。

## 技术能力
- **知识节点层级**：支持≥7级层次化知识结构提取（A-G列树状结构）
- **认知标注**：自动识别认知维度（记忆→理解→应用→分析→评价→创造）
- **关系识别**：自动生成前置/后置/关联三元关系（H-I-J列） 
- **输出格式**：符合Neo4j/Jena导入规范的CSV/Excel文件

## 使用说明

### 输入要求
- **源文档**：`docx`或`pdf`格式
- **模板文件**：`xlsx`模板，A1单元格含格式约束

### 输出格式
| 列 | 用途 | 说明 |
|---|------|------|
| A-G | 知识点层级 | 树状结构（每行一个知识点） |
| H | 前置知识 | 学习当前知识前必须掌握 |
| I | 后置知识 | 学习当前知识后可继续学习 |
| J | 关联知识 | 其他相关知识点（非依赖关系）|
| K | 标签 | 重点/难点/考点/课程思政 等 |
| L | 认知维度 | 记忆/理解/应用/分析/评价/创造 |
| M | 知识分类 | 事实性/概念性/程序性/元认知 |
| N-O | 教学属性 | 教学目标和知识点说明 |

## 核心约束规则
1. **树状结构**：每行只填写一个知识点（A-G列有且仅有一个非空）✅
2. **关系分隔符**：知识点间用英文分号`;`分隔 ✅
3. **单选约束**：认知维度/分类（L-M列）必须单选 ✅
4. **列完整性**：不得删除模板中的任何列（A-O列）✅

## 应用场景
- 从课程标准/考试大纲抽取知识点结构
- 构建教育领域知识图谱
- 自动化知识内容结构化分析
- 教学资源关联性分析

## 安装使用
```bash
# 命令行运行  
python scripts/extract_knowledge_graph.py 
  --source 课程标准.docx 
  --template assets/template.xlsx 
  --output 知识图谱.xlsx
```

## 重要提醒
所有输出文件均遵循固定的A-O列格式规范，可直接导入Neo4j、Apache Jena等图谱工具。
