## 安装相关

### Q1: 技能在 OpenCode 中找不到怎么办？

**A**: 请检查:
1. 技能是否已完整安装到 `~/.agents/skills/knowledge-graph-extractor/` 
2. 检查 SKILL.md 是否包含正确的 frontmatter (`name` 和 `description`)
3. 重启 OpenCode 使新技能加载

**验证命令**:
```bash
ls -la ~/.agents/skills/knowledge-graph-extractor/
cat ~/.agents/skills/knowledge-graph-extractor/SKILL.md | head -20
```

### Q2: 安装脚本报错说找不到命令怎么办？

**A**: 首先检查依赖是否安装:


## 使用相关

### Q3: 为什么抽取的层级不够深？

**A**: 层级不足通常是因为:
1. **文档结构不清晰**: 使用 Word 标题样式而非手动编号 
   - ❌ 避免: `1. 电路基础  2. 直流电路  2.1 欧姆定律`
   - ✅ 推荐: Word 中【标题1】→【标题2】→【标题3】...
2. **文档内容本身较浅**: 检查原文档是否有足够的层级内容
3. **层级标识不明显**: 在文档中增加更清晰的结构化表述

### Q4: 如何提高认知层级标注准确度？

**A**: 在源文档中使用标准动词引导:
- 记忆层级: "**了解**基本概念" "**知道**历史发展"
- 理解层级: "**理解**概念内涵" "**阐述**基本原理"  
- 应用层级: "**掌握**操作方法" "**会使用**工具" 或 "**能够**解决..."
- 更高层级使用相应动词...

### Q5: 文档中找不到预期的知识点怎么办？

**A**: 检查以下几点:
1. **是否使用了支持的格式**: 仅支持 `.docx` 和 `.pdf`
2. **内容是否为纯图片**: 图片文档需要先进行 OCR 识别
3. **结构化程度**: 太碎片化的文档会被识别为同一层级的多个叶子节点

**建议**: 优化文档结构后重试，层级结构越清晰识别越精确。

## 输出格式相关

### Q6: A-G列为什么每行只能填写一个知识点？

**A**: 这是关键的**树状结构约束**:
- ✅ 正确格式:
```
['电工技术基础', '', '', '', '', '', '']  # 仅A列非空
['', '安全用电常识', '', '', '', '', ''] # 仅B列非空  
['', '', '触电种类与形式', '', '', '', ''] # 仅C列非空
```
- ❌ 错误格式:
```
['电工技术', '安全用电', '触电形式', '', '', '', ''] # 一行多节点 - 禁止
```

此约束确保知识图谱系统能正确识别层级关系。

### Q7: H-I-J列的关系有什么区别?

**A**: 关系类型定义:

| 列 | 关系 | 说明 | 实例 |
|----|------|----|--------|
| H | 前置关系 | 必须先掌握才能学习当前节点 | 学 `欧姆定律` 前必须掌握 `电阻` |
| I | 后置关系 | 掌握当前节点后可学习的节点 | 学 `电阻` 后可学习 `欧姆定律` |
| J | 关联关系 | 相关但非依赖的节点 | `电流` 和 `电压` 概念关联 |

### Q8: 认知维度和知识分类有啥区别?

**A**: 两种不同维度的分类:

**认知维度** (L列): 表示学习者的**认知层次** (基于布鲁姆分类):
- 记忆→理解→应用→分析→评价→创造 （递增难度）

**知识分类** (M列): 表示知识的**类型属性**:  
- **亊实性**: 具体事实（"电阻单位是欧姆"）
- **概念性**: 概念关系（"欧姆定律公式"）
- **程序性**: 操作步骤（"如何测电阻"）
- **元认知**: 学习策略（"怎样选择测量工具"）

## 错误处理

### Q9: 报错 "FILE_FORMAT_ERROR" 怎么办？

**A**: 检查:
1. 源文档格式仅为 `.docx` 或 `.pdf`
2. Excel模板格式为 `.xlsx` (Microsoft Excel格式) 
3. 文件权限（确保可读取）

### Q10: 报错 "CELL_READ_ERROR" 有何含义？

**A**: 表示无法读取 Excel 模板的 A1 单元格，检查:
1. Excel 文件是否损坏（尝试手动打开）
2. 是否为真正的 Excel 格式（不是 TXT 重命名）
3. A1 单元格是否为空（必须包含输出格式规范）

### Q11: 报错 "HIERARCHY_INSUFFICIENT" 怎么处理？

**A**: 表示从文档中抽取不到足够的 6 级知识点，需要:
1. 上传结构更复杂、层级更多的文档
2. 确保原文档有清晰的知识结构（如章节、子章节）
3. 检查文档是否过于精简或碎片化

## 性能优化

### Q12: 大文档处理慢怎么办？

**A**: 优化策略:

1. **分割处理**:
   ```bash
   # 将大文档按章节拆分成多个较小文档分别处理
   ```

2. **优先处理关键章节**:
   - 提取文档中最关注的部分先行处理
   - 逐步扩展至全文档

3. **优化文档结构**: 
   - 使用 Word 标题样式而非手工编号
   - 保持一致的层级标识方法

### Q13: 如何确保输出文件能直接导入 Neo4j？

**A**: 验证导出文件是否满足:

1. **树状结构完整** (A-G列每行仅一非空单元格)
```bash
# 检查每行列数
awk -F',' 'BEGIN{FS=OFS=","} NR>1 {count=0; for(i=1;i<=7;i++) if($i!="") count++; if(count!=1) print "错误行", NR, "非空:", count;}' output.csv
```

2. **关系分隔符正确** (H-J列用英文分号 `;`)
3. **认知维度单选值** (L列仅为预定义六选项之一)
4. **无缺失列** (A-O列完整保留) 

**导入 Neo4j 示例**:
```cypher
LOAD CSV WITH HEADERS FROM 'file:///output.csv' AS row
CALL {
  WITH row
  WHERE row.`一级知识点` IS NOT NULL AND row.`一级知识点` <> ''
  CREATE (n1:Concept {name: row.`一级知识点`, level: '1'})
}
```

## 高级技巧

### Q14: 如何自定义模板格式?

**A**: 只要保留 A1 单元格的格式规范描述，可以调整:
1. **调整列宽** - 不改变列结构
2. **添加辅助列** - 但确保 A-O 列完整
3. **修改样式** - 不影响内容填充逻辑

### Q15: 如何处理中文/韩文等东亚语言?

**A**: 本技能全量支持多语言，特别是:
- ✅ 文档语言不限制（中/韩/日/英文档均可）
- ✅ 输出使用原字符内容（不做语言转换） 
- ✅ 标签和关系可使用中文术语
- ✅ 仅分隔符需要使用英文 `;` (此为格式规范需求)
